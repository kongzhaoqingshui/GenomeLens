# 修改位置汇总

本文档记录 GenomeLens 当前对上游源码的直接改动，方便以后对接新版上游时复刻或重新评估补丁。

## 结论

本项目为 Windows 运行和当前依赖环境，对 vendored JCVI(随包 JCVI) 做了少量直接修改。
实际运行使用的是 `jcvi-genomelens/src/jcvi/`，参考上游副本位于 `references/upstream/jcvi/src/jcvi/`。

当前与上游副本相比：

- 新增文件：`jcvi-genomelens/src/jcvi/_version.py`
- 修改文件：`jcvi-genomelens/src/jcvi/apps/base.py`
- 修改文件：`jcvi-genomelens/src/jcvi/graphics/tree.py`

`jcvi-genomelens/src/jcvi_genomelens/` 是 GenomeLens 自己的 engine wrapper(引擎包装层)，不属于上游 JCVI 补丁。

## 上游 JCVI 补丁

### `jcvi/apps/base.py`

修改位置：

- `jcvi-genomelens/src/jcvi/apps/base.py:40`
- `jcvi-genomelens/src/jcvi/apps/base.py:1122`
- `jcvi-genomelens/src/jcvi/apps/base.py:1177`
- `jcvi-genomelens/src/jcvi/apps/base.py:1228`

修改内容：

- `SIGPIPE` 设置增加 `hasattr(signal, "SIGPIPE")` 判断，避免 Windows 上没有 `SIGPIPE` 时导入失败。
- `sh()` 默认 shell(外壳) 是 `/bin/bash`，Windows 上改为 `None`，避免强依赖 bash。
- `Popen()` 同样在 Windows 上取消 `/bin/bash` executable(可执行文件) 指定。
- `which()` 在 Windows 上为未带 `.exe` 的程序名追加 `.exe` 候选项，使 `blastn`、`makeblastdb` 等命令可以解析到系统 PATH 中的 `blastn.exe`、`makeblastdb.exe`。

升级迁移建议：

- 优先检查新版 JCVI 是否已经内置这些 Windows 兼容逻辑。
- 如果没有，按同样语义迁移，不要把 shell 调用改成 `shell=False`，因为上游 `sh()`/`Popen()` 调用方仍可能传入 shell command(外壳命令) 字符串。
- 迁移后至少运行 `conda run -n genomelens python -m pytest jcvi-genomelens/tests` 和一次真实 `check --json`。

### `jcvi/graphics/tree.py`

修改位置：

- `jcvi-genomelens/src/jcvi/graphics/tree.py:8`

修改内容：

- 上游只导入 `ete4.Tree`。
- GenomeLens 当前依赖环境使用 `ete3`，所以增加 fallback(回退)：优先导入 `ete4.Tree`，失败后导入 `ete3.Tree`。

升级迁移建议：

- 如果新版环境改为 `ete4`，可以重新评估是否删除该 fallback(回退)。
- 如果继续使用 `ete3`，保留这段兼容导入。

### `jcvi/_version.py`

修改位置：

- `jcvi-genomelens/src/jcvi/_version.py:1`

修改内容：

- 新增版本元数据文件，声明 vendored JCVI(随包 JCVI) 版本为 `1.6.5`。
- 该文件用于 GenomeLens probe(探测) 与运行摘要报告，不改变 JCVI 分析逻辑。

升级迁移建议：

- 对接新版 JCVI 时同步更新 `__version__`。
- 该版本号应与 `jcvi_genomelens.probe` 输出中的 `jcvi_upstream_version` 保持一致。

## 新版对接流程

1. 将新版上游 JCVI 放入 `references/upstream/jcvi/`。
2. 将新版源码同步到 `jcvi-genomelens/src/jcvi/`。
3. 按本文档复刻 `apps/base.py`、`graphics/tree.py` 和 `_version.py` 的补丁，或确认新版已经不再需要。
4. 运行内容差异检查：

```powershell
@'
from pathlib import Path
left = Path('references/upstream/jcvi/src/jcvi')
right = Path('jcvi-genomelens/src/jcvi')
left_files = {p.relative_to(left) for p in left.rglob('*') if p.is_file() and '__pycache__' not in p.parts}
right_files = {p.relative_to(right) for p in right.rglob('*') if p.is_file() and '__pycache__' not in p.parts}
print('ADDED', sorted(right_files - left_files))
print('REMOVED', sorted(left_files - right_files))
print('CHANGED', [p for p in sorted(left_files & right_files) if (left / p).read_bytes() != (right / p).read_bytes()])
'@ | python -
```

5. 确认差异仍然只包含本文档记录的补丁点。
6. 运行完整验证：

```powershell
conda run -n genomelens python -m pytest python/tests jcvi-genomelens/tests
Push-Location integrations\haiant_plugin
$env:PYTHONPATH = (Resolve-Path .\src).Path
conda run -n genomelens python -m pytest tests
Pop-Location
conda run -n genomelens python -m genomelens.cli.main check --json
```
