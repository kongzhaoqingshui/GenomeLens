# 项目介绍

GenomeLens 是面向 Windows-first 交付的比较基因组学平台原型。它把输入校验、数据预处理、JCVI 分析、JCVI 绘图和结果归档拆成清晰的 shell(外壳) 与 engine(引擎) 边界，尽量减少用户手工拼接 BLAST+、JCVI 命令和图形参数的负担。

## 完整目标

项目完整目标是：从 2 到任意多个物种的 GFF/GTF(annotation，注释文件) 与 FASTA(基因组序列) 开始，完成全流程参数预设，自动执行 JCVI 分析与绘图能力，最终生成多物种、可美化、可发表的共线性图。

完整目标包含：

- 多物种 `species[]` 输入模型。
- 每个物种的 GFF/GTF + FASTA 预处理。
- 2 到任意物种的 pairwise JCVI 分析矩阵。
- anchors(锚点)、blocks(共线性区块)、ortholog(同源基因) 等中间结果聚合。
- 自动生成和人工可调的 seqids/layout 图形布局。
- dotplot(点图)、synteny figure(共线性图)、karyotype(核型共线性图) 等 JCVI 图形能力。
- 最终美化版多物种共线性图导出。

## 当前版本能力

当前版本已经实现：

- `analyze mcscan <input> <output>` 自动目录入口。
- 2 个物种时运行真实双物种 BLAST+ + JCVI 流程。
- 3 个以上物种时自动拆成 all-vs-all pairwise(全组合两两比较) 子任务。
- GFF/GTF 与 FASTA 按物种预处理为 JCVI 所需 BED/CDS，可与已准备好的 BED+CDS/PEP 物种混合输入。
- 默认 `graphics_synteny` workflow(工作流) 同时输出 `dotplot.svg` 与 `synteny.svg`。
- 独立 `graphics_dotplot`、`graphics_karyotype`、`catalog_ortholog` workflow(工作流)。
- 多物种运行后，把各 pairwise 共线性边聚合成一张跨全部物种的全局核型总图（`graphics_karyotype_global`）。
- 配置文件参与工具链路径、线程数、block size 和输出格式预设。
- HAIant plugin(智然体插件) 参数映射与打包交付。

## 尚未完成

当前多物种能力是 pairwise 编排、汇总，以及由各 pairwise 共线性边聚合出的一张全局核型总图。这张总图使用默认 layout/seqids 与默认配色，还不是经过自动优化的最终美化版。尚未完成：

- 全局总图的美化：自动 layout/seqids 优化、配色与排版精修。
- 多物种区块合并、排序和过滤。
- 机器学习评分与候选优先级排序。

## 模块边界

- `platform/` 是 GenomeLens 平台核心(Platform Core) 雏形，负责用户命令、输入校验、预处理、工具链定位、manifest(清单) 写入和结果归档；包名仍为 `genomelens`。
- `engines/jcvi/` 是当前唯一正式 engine(引擎)，内置 vendored JCVI(随包 JCVI)，负责真实分析计算与绘图；未来其他比较基因组学引擎与它平级，置于 `engines/` 下。
- `integrations/haiant_plugin/` 是 HAIant adapter(智然体适配器)，负责把 `params.json` 转换为 GenomeLens CLI 参数。
- `toolchains/` 是本地 runtime cache(运行时缓存)，不作为源码提交。

## 交付原则

源码仓库只保存可审查、可维护的工程文件。大型离线工具链、构建输出和插件压缩包通过 release attachments(发布附件) 或独立 artifact store(制品库) 交付，避免 Git 仓库承载大文件。
