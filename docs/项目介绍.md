# 项目介绍

## 1.X 定位

GenomeLens 是面向 Windows-first 交付的本地比较基因组学平台，当前版本 **v0.9.20**，正在向 **1.X** 架构演进。

在 1.X 阶段，GenomeLens 整合 **JCVI 及其扩展的全部能力**，并通过两种互补方式交付：

1. **一站式工作流（One-Stop Workflow）**：面向最常见场景，一个命令完成从输入发现、预处理、比对、MCscan 到出图的全链路。内部保留专用 runner 和优化空间，**不是子模块的简单拼接**。
2. **可编排子模块（Composable Sub-Module）**：每个 JCVI 能力以显式输入/输出端口暴露，可独立运行，也可通过脚本、批处理、插件或 GUI 积木式组合。

在此之上，GenomeLens 引入自研优化（如原生共线性渲染器），并计划在后续版本中逐步接入更多现代比较基因组学分析方法。

核心目标：在 Windows 上**本地、轻量、快捷、高效、真实**地完成 JCVI 相关生信分析。

## 当前版本能力（v0.9.20）

### 一站式工作流

| workflow_id | 场景 | 说明 |
|---|---|---|
| `pairwise_synteny` | 双物种共线性 | MCscan + dotplot + synteny |
| `multi_species_synteny` | 多物种 all-vs-all | 拆分为 pairwise 子任务并聚合为全局核型总图 |
| `reference_vs_targets` | 参考物种 vs 多个目标物种 | pairwise + 以目标基因为中心的局部共线性聚合 |
| `histogram_plot` | 数值分布直方图 | 纯 matplotlib 渲染 |
| `heatmap_plot` | 矩阵热图 | 直接渲染 CSV 矩阵 |

入口命令：

```powershell
GenomeLens.exe analyze workflow <workflow_id> <input> <output> [options] --force
```

### 可编排子模块

已注册 10 个 JCVI 子模块，覆盖同源搜索、可视化、局部共线性：

- `jcvi.mcscan_pairwise`：双物种比对、锚点扫描、block 计算。
- `jcvi.catalog_ortholog`：基于共线性 + RBH 识别双向同源基因。
- `jcvi.graphics_dotplot`：共线性点图。
- `jcvi.graphics_synteny`：共线性对齐图。
- `jcvi.graphics_karyotype`：核型共线性图。
- `jcvi.graphics_histogram`：数值分布直方图。
- `jcvi.graphics_heatmap`：矩阵热图。
- `jcvi.graphics_karyotype_global`：多物种全局核型总图。
- `jcvi.local_synteny`：双物种局部共线性。
- `jcvi.local_synteny_multi`：多物种局部共线性聚合。

入口命令：

```powershell
GenomeLens.exe analyze submodule <module_id> --input-ports '<json>' --output-dir <dir> [options] --force
```

### 统一能力发现

```powershell
GenomeLens.exe workflow list --json
GenomeLens.exe workflow describe <id> --json
GenomeLens.exe workflow validate --submodule <module_id> --ports '<json>' --json
```

### 统一请求格式

CLI、插件、GUI 共用 `AnalysisRequest` JSON，`task_kind` 区分：

- `analysis`：旧模式（`analyze run`）。
- `one_stop`：一站式工作流（`analyze workflow <workflow_id>`）。
- `sub_module`：单个子模块（`analyze submodule <module_id>`）。
- `composition`：可视化组合（未来实现）。

### 其他能力

- GFF/GTF + FASTA 自动预处理为 BED/CDS；可与 BED+CDS/PEP 混用。
- 比对后端可选：BLAST+ / LAST / Diamond。
- 2 个物种走真实双物种流程；3 个及以上自动拆分为 all-vs-all pairwise。
- 原生 `local_synteny_renderer` 作为 JCVI 默认渲染器的替代选项。
- 配置文件参与工具链路径、线程数、block size 和输出格式预设。
- HAIant plugin（智然体插件）参数映射与打包交付。
- 先行 GUI 版本 **JCVI meow**；GUI 通过 `AnalysisRequest` JSON 与平台通信。

## 模块边界

- `platform/`：平台核心，负责 CLI、输入校验、预处理、工具链定位、manifest 写入、engine 调用、结果归档，以及子模块/一站式注册表。
- `engines/jcvi/`：当前唯一正式引擎，内置 vendored JCVI，只暴露 `probe` 和 `run` 两个入口。
- `engines/`：引擎层目录，未来其他比较基因组学引擎与它平级接入。
- `integrations/haiant_plugin/`：HAIant 插件适配器，负责把 `params.json` 转换为 GenomeLens CLI 参数。
- `gui/`、`agents/`：路线图预留位置；GUI 已实现先行版。
- `toolchains/`：本地运行时缓存，不进入 Git 跟踪。

跨层通信只通过 `jcvi_engine_manifest.json` 与 `engine_run_summary.json`。

## 路线图

- **短期**：补齐 JCVI 图形与 QC 能力；实现 `composition` 可视化编排执行。
- **中期**：扩展比较基因组学增强能力（Ks 分析、QUOTA-ALIGN、SynFind、系统发育树等）。
- **长期**：以插件或独立引擎形式引入 Assembly、Annotation、Variation 等现代比较基因组学方法，逐步超越 JCVI 的部分实现。

## 交付原则

源码仓库只保存可审查、可维护的工程文件。大型离线工具链、构建输出和插件压缩包通过 release attachments 或独立制品库交付，避免 Git 仓库承载大文件。
