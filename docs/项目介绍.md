# 项目介绍

## 1.X 定位

GenomeLens 是面向 Windows-first 交付的本地比较基因组学平台。

在 1.X 阶段，GenomeLens 将整合 **JCVI 及其扩展的全部能力**，并提供两种使用方式：

1. **一站式工作流**：一个命令完成从输入发现、预处理、比对、MCscan 到出图的全链路（如 `analyze mcscan jcvi`）。
2. **可编排子模块**：按需调用单个 JCVI 能力或外部 JSON 请求，方便脚本、批处理、插件和 GUI 集成。

在此之上，GenomeLens 引入自研优化（如原生共线性渲染器），使研究者能在 Windows 上**本地、轻量、快捷、高效、真实**地完成 JCVI 相关生信分析，并计划在后续版本中逐步超越 JCVI 的部分实现与算法，引入更多现代且优质的比较基因组学分析方法。

## 当前版本能力（v0.9.20）

- `analyze mcscan jcvi <input> <output>` 自动目录入口。
- GFF/GTF + FASTA 自动预处理为 JCVI 所需 BED/CDS；可与已准备好的 BED+CDS/PEP 混用。
- 2 个物种时运行真实双物种 BLAST+ + JCVI 流程。
- 3 个及以上物种时自动拆分为 all-vs-all pairwise 子任务，并汇总为全局核型总图。
- workflow 覆盖：`graphics_synteny`、`graphics_dotplot`、`graphics_karyotype`、`local_synteny`、`graphics_histogram`、`catalog_ortholog`。
- 以参考物种目标基因为中心的 `local_synteny` 局部共线性链路。
- 原生 `local_synteny_renderer` 作为 JCVI 默认渲染器的替代选项。
- 配置文件参与工具链路径、线程数、block size 和输出格式预设。
- HAIant plugin（智然体插件）参数映射与打包交付。
- 先行 GUI 版本 JCVI meow。

## 模块边界

- `platform/`：平台核心，负责 CLI、输入校验、预处理、工具链定位、manifest 写入、engine 调用与结果归档。
- `engines/jcvi/`：当前唯一正式引擎，内置 vendored JCVI，只暴露 `probe` 和 `run` 两个入口。
- `engines/`：引擎层目录，未来其他比较基因组学引擎与它平级接入。
- `integrations/haiant_plugin/`：HAIant 插件适配器，负责把 `params.json` 转换为 GenomeLens CLI 参数。
- `gui/`、`agents/`：路线图预留位置。
- `toolchains/`：本地运行时缓存，不进入 Git 跟踪。

## 路线图

- **短期**：继续补齐 JCVI 图形与 QC 能力（Stats/Summary/Depth、Heatmap、BLAST Dotplot、Tandem Genes 等）。
- **中期**：扩展比较基因组学增强能力（Ks 分析、QUOTA-ALIGN、SynFind、系统发育树等）。
- **长期**：以插件或独立引擎形式引入 Assembly、Annotation、Variation 等现代比较基因组学方法，逐步超越 JCVI 的部分实现。

## 交付原则

源码仓库只保存可审查、可维护的工程文件。大型离线工具链、构建输出和插件压缩包通过 release attachments 或独立制品库交付，避免 Git 仓库承载大文件。
