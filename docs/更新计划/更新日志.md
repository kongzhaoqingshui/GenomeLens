- **HAIant 插件架构最终调整**：所有单功能插件保持完全独立；`gljcvi-auto` 统一自动流插件固定对应 `analyze mcscan jcvi` 一键分析流程。

  - 删除 `scripts/build_haiant_plugin.ps1`、`scripts/build_gljcvimcscan_center.ps1`。
  - 从 `_core.py` 与 `__init__.py` 中移除 `discover_mcscan_home`、`genomelens_shell_path`、`runtime_executable`、`GLJCVIMCSCAN_HOME`、`GENOMELENS_PLUGIN_RUNTIME`、`SUPPORTED_WORKFLOWS` 等旧 helper 与常量。
  - 精简并更新插件测试，移除重型中心相关用例。
- **核心平台产物重命名**：Windows 平台 PyInstaller 产物从 `GenomeLens-runtime.exe` 统一改为 `GenomeLens.exe`；同步更新 `platform/packaging/pyinstaller/genomelens.spec`、`platform/src/genomelens/toolchain/runtime/platform_names.py`、`.github/workflows/release-smoke.yml`、CLI 入口壳文档以及所有 HAIant 插件配置/示例/测试中的可执行文件示例路径。

## [0.9.15] - 2026-06-21

### 新增

- **JCVI Heatmap 独立绘图**：新增 `graphics_heatmap` engine workflow 与 `plot heatmap` CLI 命令，支持从矩阵 CSV 直接生成热图，并可选列分组、行分组、水平色条和自定义 colormap。
- **JCVI Histogram 独立绘图**：新增 `graphics_histogram` engine workflow 与 `analyze mcscan jcvi graphics_histogram` CLI 入口，支持读取一个或多个数值文件生成 histogram（直方图），并提供 `--histogram-columns`、`--histogram-bins`、`--histogram-base`、`--histogram-facet`、`--histogram-fill` 等参数。
- **plot-only request / runner 链路**：Heatmap 与 Histogram 不再强制要求物种目录输入，而是通过 method-specific request、独立 runner 和 engine manifest 走完整的 summary / artifact 归档流程。

### 变更

- **Histogram 绘图后端**：由于当前 `genomelens` 环境未捆绑 `Rscript`，Histogram 改用 Python/matplotlib 后端实现，CLI 参数语义保持不变。

### 测试

## [0.9.14] - 2026-06-21

### 重构

- **自动优化参数分组**：四个 GenomeLens 添加的自动优化开关（`optimize_figsize`、`rewrite_layout_links`、`fix_karyotype_label_overlap`、`trim_cross_chromosome_blocks`）从 `local_synteny` / engine manifest `options` 的扁平结构中抽出，统一放到 `local_synteny.auto_optimization` / `options.auto_optimization` 嵌套结构下，便于区分原生 JCVI 参数与本工具扩展参数。
  - 配置模型新增 `AutoOptimizationDefaults` dataclass。
  - `McscanMethodConfig`、`McscanRequest`、`WorkflowOptions` 均改为持有 `auto_optimization: dict[str, bool]`。
  - CLI 与配置合并逻辑同步改为 `_auto_optimization_dict()` 集中处理。

### 变更

- **重命名全局核型标签优化开关**：`fix_karyotype_label_overlap` 重命名为 `optimize_karyotype_labels`，更准确表达其用途（优化全局核型图轨道标签位置）。对应 CLI flag 由 `--fix-karyotype-label-overlap` 改为 `--optimize-karyotype-labels`。
- **收紧标签间距**：镜像核型 renderer 的 `Track.draw()` 默认 `vpad` 从 `0.09` 降至 `0.05`，标签与轨道间距更紧凑。

### 不兼容说明

- 本版本不保留旧版配置/API 兼容：`jcvi.config.json` 中原扁平的四个优化键、engine manifest 中的扁平键、旧 CLI flag 均已移除。
