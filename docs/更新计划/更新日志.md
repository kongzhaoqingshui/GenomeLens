# 更新日志

## [1.0.1] - 已撤回（Yanked）

> ⚠️ 该版本曾短暂发布，但因不符合 SemVer 规范且核心功能尚未达到稳定版标准，已回退到 `0.9.x` 预发布阶段。请勿使用。

### 撤回原因

- 版本号策略过于激进：当前核心功能仍在 0.9.x 阶段补齐中，尚未满足对外承诺的 `v1.0.0` 稳定版标准。
- 根据 [SemVer 2.0](https://semver.org/lang/zh-CN/)，`v1.0.1` 应基于已发布的 `v1.0.0` 仅做向后兼容的问题修复，但 `v1.0.0` 本身尚未正式发布。

### 后续处理

- GitHub Releases `v1.0.0`、`v1.0.1` 已删除。
- Git 标签 `v1.0.0`、`v1.0.1` 已删除。
- 本地归档镜像已重命名为 `GenomeLens-pre-0.9.0-mirror.git`。
- 项目当前处于 `0.9.x` 预发布阶段，`v1.0.0` 将在核心功能全部补齐后正式发布。

### 关于本地 Git 历史归档

项目曾对 Git 历史做过两次本地归档，完整原始历史均保留在本地镜像中，GitHub 上仅保留干净起点：

1. **第一次归档**：从探索性内部开发阶段转向正式结构化开发阶段，将早期杂乱历史归档。
2. **第二次归档**：因过早将尚未完善的版本标记为 `v1.0.0`/`v1.0.1` 并发布，为避免版本号与开发阶段混淆，将此次发布前的完整历史再次归档，并在 GitHub 上以 `v0.9.0` 预发布阶段作为新起点。

## [0.9.6] - 2026-06-19

### 新增

- **CLI 紧凑进度报告**：`analyze mcscan jcvi` 运行时展示单行进度/状态行，包括当前步骤、物种对、已完成比例，控制台不再静默地只输出日志。
  - 新增 `CompactProgressRenderer` 与 `SignalBus` 事件集成。
  - 多物种运行时按物种对进度更新，失败时状态行同步标红。

### 改进

- **出图跨染色体裁切回退**：当 `trim_cross_chromosome_blocks` 裁切后的 blocks 为空或无法被 JCVI 渲染时，自动回退到原始 blocks，避免生成空白图。
- **出图布局链接回退**：`rewrite_layout_links` 在无法构成有效链时保留原始边。

### 修复

- **Pyright 类型错误**：修复 `cli/ui.py` 进度事件 `payload.get("index")` 的 `object | int` 类型窄化，以及 `print_help` 方法签名与基类不兼容的问题。

---

## [0.9.7] - 2026-06-19

### 新增

- **JCVI 子任务 CLI**：`analyze mcscan jcvi <subtask>` 支持直接调用 `graphics_synteny`、`graphics_dotplot`、`local_synteny` 等子任务。
- **多物种局部共线性总图**：`reference_vs_targets` 模式下，当物种数 ≥3 且提供 `--target-genes` 时，自动生成多物种局部共线性汇总图。

### 改进

- **请求模块结构重组**：`genomelens.analysis` 下与请求处理相关的模块统一迁入 `genomelens.analysis.requests/` 子目录，结构更清晰。
  - `request_models.py` → `requests/models.py`
  - `request_schema.py` → `requests/schema.py`
  - `request_loader.py` → `requests/loader.py`
  - `request_normalizer.py` → `requests/normalizer.py`
  - `normalization/` → `requests/normalization/`
  - 旧路径的向后兼容 shim 已在本版本中移除，所有内部引用已迁移到新路径。
- **Engine workflow 辅助函数去重**：8 个 workflow 中重复的 `_assert_ok` 提取到 `workflows/common.py`。
- **代码归属整理**：
  - `mcscan_summary.py` 从 `analysis/methods/` 移动到 `core/`，更贴合其作为 RunSummary 扩展的职责。
  - `parsers.py` 从 `genomelens/` 根目录移动到 `utils/`，作为通用工具函数。
- **清理无用文件**：删除 `analysis/methods/mcscan.py`（旧适配器，已被 `mcscan_plugin` 替代）和 `recc_need.py`（占位文件）。

### 修复

- **plot_optimization 命名歧义**：将仅返回 body 的 `_split_block_highlight` 重命名为 `_strip_highlight_prefix`，避免与返回 `(highlight, body)` 的同名函数混淆。

## [0.9.5] - 2026-06-18

### 新增

- **控制台静默模式**：控制台默认只输出初始启动句段与警告/错误，完整运行日志继续写入 `run.log`。
  - `ConciseConsoleFilter` 仅放行 `Starting GenomeLens` 前缀的 INFO 消息。
  - WARNING/ERROR/CRITICAL 消息始终输出到控制台。

### 改进

- **JCVI 出图尺寸取整**：`plot_optimization` 的 `suggest_figsize` 返回整数 `WxH` 尺寸，适配 JCVI 对 figsize 的格式要求。

### 修复

- **Pyright 类型错误**：
  - `manifest_loader.py` 使用 `_int` 辅助函数解析 `schema_version`，避免 `int(object | Literal[1])` 类型不匹配。
  - `local_synteny_aggregate.py` 对 `artifacts.get("figures")` 使用 `cast(list[Any], ...)`，修复 `"object" is not iterable`。
  - 在 `catalog_ortholog.py`、`graphics_dotplot.py`、`graphics_karyotype.py`、`local_synteny.py`、`mcscan_pairwise.py` 的 workflow 入口增加 `query`/`subject` 非空守卫，消除 `reportOptionalMemberAccess`。

## [0.9.4] - 2026-06-18

### 新增

- **多物种局部共线性汇总图**：`reference_vs_targets` 模式下，以参考物种为中心，聚合多个目标物种的局部共线性结果生成 `multi_species_local.*` 汇总图。
- **出图自动优化控制**：engine 新增 `optimize_figsize`、`rewrite_layout_links`、`trim_cross_chromosome_blocks` 三个优化开关，可在 `jcvi.config.json` 或 CLI 中控制。
- **CLI 新增 plot 优化参数**：`analyze mcscan jcvi` 支持 `--optimize-figsize`、`--rewrite-layout-links`、`--trim-cross-chromosome-blocks`。

### 改进

- **配置文件默认值扩展**：`jcvi.config.json` 初始化时暴露 plot 优化相关默认项。

## [0.9.3] - 2026-06-18

### 新增

- **JCVI 子任务 CLI**：`analyze mcscan jcvi <subtask>` 支持直接调用底层 JCVI workflow，例如 `graphics_dotplot`、`graphics_synteny`。
- **子任务 workflow 强制覆盖**：子命令本身决定 workflow，即使传入 `--jcvi-workflow` 也不会被覆盖，避免命令意图与参数冲突。

### 改进

- **帮助页索引扩展**：`help analyze mcscan` 列出可用子任务页；`help analyze mcscan jcvi <subtask>` 输出子任务专用帮助。

## [0.9.2] - 2026-06-18

### 新增

- **`analyze run <request.json>` 外部配置入口**：恢复通过完整 `AnalysisRequest` JSON 驱动分析流程。
- **`analyze template` 与 `analyze schema`**：分别输出 mcscan 请求示例与 JSON schema，便于外部系统生成请求。
- **`AnalysisRequest` JSON schema**：稳定协议包含 `input`、`output`、`config`、`options`、`method_config` 等字段。

### 集成

- **智然体插件入口更新**：插件从直接调用 `analyze mcscan` 改为写入 `genomelens_request.json` 并调用 `analyze run`；仅暴露 `graphics_synteny` workflow。

## [0.9.1] - 2026-06-18

### 新增

- **结构化 `run.log`**：shell 顶层、pairwise 子任务、engine 工作流步骤统一记录 `task_started` / `task_finished` / `task_failed`，字段包含 `task_id`、`step`、`status`、`elapsed_ms`、`context`。
- **CLI 诊断参数**：`analyze mcscan jcvi` 支持 `--verbose` 与 `--log-level {DEBUG,INFO,WARNING,ERROR}`；配置项 `log_level` 可回退，默认 `INFO`。
- **失败上下文**：engine 步骤失败时记录最后执行的 `argv`、`cwd`、`stderr` 摘要。

### 改进

- **多物种嵌套日志**：顶层 `run.log` 不被每个 pairwise 子任务的 `setup_logging` 清除；每个 pair 目录仍保留自己的 `logs/run.log`。
