# 更新日志

## [1.0.0] - 尚未发布

### 计划内容

- 双物种真实 JCVI 端到端共线性分析

- 多物种以目标基因为中心的局部共线性分析

### 说明

v1.0.0 是项目对外承诺的第一个稳定版本。当前仍在 0.9.x 阶段迭代补齐核心能力，待上述功能全部完成后正式发布 v1.0.0。

## [0.9.14] - 2026-06-21

### 重构

- **自动优化参数分组**：四个 GenomeLens 添加的自动优化开关（`optimize_figsize`、`rewrite_layout_links`、`fix_karyotype_label_overlap`、`trim_cross_chromosome_blocks`）从 `local_synteny` / engine manifest `options` 的扁平结构中抽出，统一放到 `local_synteny.auto_optimization` / `options.auto_optimization` 嵌套结构下，便于区分原生 JCVI 参数与本工具扩展参数。
  - 配置模型新增 `AutoOptimizationDefaults` dataclass。
  - `McscanMethodConfig`、`McscanRequest`、`WorkflowOptions` 均改为持有 `auto_optimization: dict[str, bool]`。
  - CLI 与配置合并逻辑同步改为 `_auto_optimization_dict()` 集中处理。

### 变更

- **重命名全局核型标签优化开关**：`fix_karyotype_label_overlap` 重命名为 `optimize_karyotype_labels`，更准确表达其用途（优化全局核型图轨道标签位置）。对应 CLI flag 由 `--fix-karyotype-label-overlap` 改为 `--optimize-karyotype-labels`。
- **收紧标签间距**：镜像核型 renderer 的 `Track.draw()` 默认 `vpad` 从 `0.09` 降至 `0.05`，标签与轨道间距更紧凑。

### 不兼容说明

- 本版本不保留旧版配置/API 兼容：`jcvi.config.json` 中原扁平的四个优化键、engine manifest 中的扁平键、旧 CLI flag 均已移除。

## [0.9.8] - 2026-06-19

### 新增

- **全局核型总图布局优化**：`graphics_karyotype_global` 支持 `rewrite_layout_links` 与 `optimize_figsize`，在保守重排轨道顺序的同时把 `--figsize`/`--dpi` 传入 JCVI，并把 `rewritten_layout_edges`、`rewritten_track_order`、`optimized_figsize` 写入 artifacts。
- **多物种局部共线性图布局优化**：`local_synteny_multi` 支持 `rewrite_layout_links`（链式 edge 改写）与 `optimize_figsize`（自动推导尺寸），并把 `rewritten_layout`、`rewritten_layout_edges`、`optimized_figsize`、`multi_species_local_layout` 写入 artifacts。
- **plot 优化开关全面透传**：`build_global_karyotype_manifest()` 与 `build_multi_local_synteny_manifest()` 补齐 `figsize`、`dpi`、`optimize_figsize`、`rewrite_layout_links` 参数。

### 改进

- **移除请求模块向后兼容 shim**：`genomelens.analysis.request_{loader,models,normalizer,schema}` 已删除，所有内部引用统一使用 `genomelens.analysis.requests.*`。
- **CLI 进度条可见性**：Workbench 模式下始终显示紧凑进度报告，避免子任务运行时控制台静默。

### 测试

- 新增 `test_multi_species_plot_workflows.py` 覆盖 multi-local / global 布局改写与 figsize 推导。
- 扩展 `test_engine_run.py` 与 `test_cli.py` 验证 manifest 选项、最终 layout、`--figsize` argv 及优化 artifacts。

### 新增

  - `request_schema.py` → `requests/schema.py`

  - `request_normalizer.py` → `requests/normalizer.py`
  - `normalization/` → `requests/normalization/`

- **Engine workflow 辅助函数去重**：8 个 workflow 中重复的 `_assert_ok` 提取到 `workflows/common.py`。
- **代码归属整理**：
  - `mcscan_summary.py` 从 `analysis/methods/` 移动到 `core/`，更贴合其作为 RunSummary 扩展的职责。
  - `parsers.py` 从 `genomelens/` 根目录移动到 `utils/`，作为通用工具函数。
- **清理无用文件**：删除 `analysis/methods/mcscan.py`（旧适配器，已被 `mcscan_plugin` 替代）和 `recc_need.py`（占位文件）。

### 修复

- **plot_optimization 命名歧义**：将仅返回 body 的 `_split_block_highlight` 重命名为 `_strip_highlight_prefix`，避免与返回 `(highlight, body)` 的同名函数混淆。

## [1.0.1] - 已撤回（Yanked）

> ⚠️ 该版本曾短暂发布，但因不符合 SemVer 规范且核心功能尚未达到稳定版标准，已回退到 `0.9.x` 预发布阶段。请勿使用。

### 撤回原因

- 版本号策略过于激进：当前核心功能仍在 0.9.x 阶段补齐中，尚未满足对外承诺的 `v1.0.0` 稳定版标准。
- 根据 [SemVer 2.0](https://semver.org/lang/zh-CN/)，`v1.0.1` 应基于已发布的 `v1.0.0` 仅做向后兼容的问题修复，但 `v1.0.0` 本身尚未正式发布。

### 后续处理

- GitHub Releases `v1.0.0`、`v1.0.1` 已删除。
- Git 标签 `v1.0.0`、`v1.0.1` 已删除。
- 本地归档镜像已重命名为 `GenomeLens-pre-0.9.0-mirror.git`。
- 项目当前处于 `0.9.x` 预发布阶段，`v1.0.0` 将在核心功能全部补齐后正式发布。

### 关于本地 Git 历史归档

项目曾对 Git 历史做过两次本地归档，完整原始历史均保留在本地镜像中，GitHub 上仅保留干净起点：

1. **第一次归档**：从探索性内部开发阶段转向正式结构化开发阶段，将早期杂乱历史归档。
2. **第二次归档**：因过早将尚未完善的版本标记为 `v1.0.0`/`v1.0.1` 并发布，为避免版本号与开发阶段混淆，将此次发布前的完整历史再次归档，并在 GitHub 上以 `v0.9.0` 预发布阶段作为新起点。
