# 更新日志

## [Unreleased]

### 架构

- **子模块职责重整（计算/渲染解耦）**：`jcvi.mcscan_pairwise` 与 `jcvi.catalog_ortholog` 合并为单一计算模块 `jcvi.pairwise`，ortholog 目录改由参数 `emit_ortholog=true` 选择输出；子模块数 10 → 9。
- **渲染模块去计算**：`graphics_dotplot/synteny/karyotype`、`local_synteny` 不再在缺产物时静默重跑 pairwise（移除 `fallback_runner=run_pairwise`）；`ensure_pairwise_artifacts` 缺基础产物时抛 `MissingPairwiseArtifactsError`。
- **onestop 两回合编排**：`analyze workflow synteny` 改为先在独立子目录跑 `pairwise` 计算产出共线性基础产物，再经 `precomputed_artifacts` 注入渲染回合（`command_mapping.requires_precomputed_pairwise` 判定计算入口与渲染回合）。
- **子模块契约收紧**：移除 `layout` ⟵ `karyotype_layout` 输入端口的 `or` 兜底死别名；直方图参数严格按 `histogram_` 前缀读取，不再兼容无前缀写法。
- **平台任务模型 V2**：公开请求收束为 `WorkflowRequest`，内部执行收束为 `ExecutionPlan`，JCVI engine manifest 升级为 schema v3；旧 `AnalysisRequest`、`method_config`、`task_kind` 等字段不再作为公开协议。
- **统一执行编排**：多物种 all-vs-all、reference-vs-targets、plot-only 等路径统一经 `WorkflowPlanner` / `PlanExecutor` 展开与执行，旧多物种 runner 编排路径已移除。
- **结果摘要 V3**：`RunSummary` 扩展信息进入 `extensions`，复合任务子运行进入 `child_runs`，产物统一进入 `artifact_index`。
- **HAIant 插件协议同步**：常规功能插件改为生成 `WorkflowRequest v2`，原子子模块插件改为调用 `analyze submodule`，不再生成旧 `task_kind=sub_module` 请求。

### 文档

- 用户文档改为 `WorkflowRequest JSON.md`，同步更新用户手册、项目介绍、子模块/组合说明、能力接入规则和 HAIant 插件文档。

## [1.0.0-preview-1] - 2026-06-25

> 第一个 1.0.0 预览版本。标志着 GenomeLens 进入 1.0 稳定版发布前的最后验证阶段。

### 新增

- **打包产物纳入 Cython 加速扩展**：`jcvi.formats.cblast` 与 `jcvi.assembly.chic` 在打包 pipeline 中编译并嵌入 frozen engine，运行模式升级为 `accelerated`。
- **跨轨补充线（skip link）几何修复**：多物种局部共线性跨轨道断链时的浅灰虚线改为沿正常连线几何穿过中间轨道，不再绕大圈。
- **连线宽度算法对齐 JCVI**：原生局部共线性 ribbon 宽度改用基因全长足迹，与 JCVI `Shade` 行为一致。

### 重构

- **更新日志结构优化**：0.9.x 系列全部版本历史归档至 `docs/历史/更新/0.9系列/更新日志_0.9.x.md`，主更新日志仅保留未发布改动与 1.0 预览版条目。

### 修复

- **PyInstaller 动态导入遗漏**：通过 `collect_submodules("jcvi_genomelens")` 收集全部 workflow 子模块，修复打包后 `ModuleNotFoundError: jcvi_genomelens.workflows.graphics`。
- **chic 扩展缺失容错**：新增 `jcvi.assembly.chic_fallback`，未编译扩展时引擎仍可运行在 `core` 模式。
