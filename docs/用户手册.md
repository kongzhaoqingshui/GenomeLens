# GenomeLens 用户手册

> 面向最终用户的快速参考。开发细节见 `docs/开发手册/README.md`，子模块端口明细见 `docs/使用方法/子模块手册.md`。

## 1. 环境检查

```powershell
GenomeLens.exe check
GenomeLens.exe check -j
GenomeLens.exe check --install-missing
```

开发环境：

```powershell
conda activate genomelens
python -m genomelens.cli.main check
```

## 2. 输入目录

把同一物种的文件放在同一个目录，并保持 basename 一致。

### BED + CDS/PEP

```text
input/
  speciesA.bed
  speciesA.cds
  speciesB.bed
  speciesB.cds
```

也支持蛋白序列：`.pep`、`.pep.fa`、`.faa`。

### GFF/GTF + FASTA

```text
input/
  speciesA.gff3
  speciesA.fa
  speciesB.gff3
  speciesB.fa
```

同一目录可以按物种混用两类输入。若同一物种同时提供两类文件，自动发现会优先使用 `BED + CDS/PEP`。

## 3. 两种使用方式

GenomeLens 1.X 把软件能力拆成两类：

| 维度 | 一站式工作流 | 可编排子模块 |
|---|---|---|
| 目标场景 | 常见端到端分析 | 单能力、批量脚本、GUI 积木 |
| 入口命令 | `analyze workflow <id> ...` | `analyze submodule <id> ...` |
| 输入 | 目录或主文件 | 显式端口绑定 JSON |
| 优化 | 专用 runner，保留内部优化空间 | 无隐式拼接，按端口组合 |
| 等价子模块 | 元数据中说明，仅供参考 | 自身即最小单元 |

### 3.1 一站式工作流

适合最常见的“从输入到出图”场景，一个命令跑完完整链路。

```powershell
# 双物种共线性（默认 synteny + dotplot）
GenomeLens.exe analyze workflow pairwise_synteny input output --force

# 多物种 all-vs-all 并汇总全局核型总图
GenomeLens.exe analyze workflow multi_species_synteny input output --force

# 参考物种 vs 多个目标物种的局部共线性
GenomeLens.exe analyze workflow reference_vs_targets input output `
  --reference subject --target-genes AT1G01010 --up 20 --down 20 --force

# 数值分布直方图
GenomeLens.exe analyze workflow histogram_plot numbers.txt output --force

# 矩阵热图
GenomeLens.exe analyze workflow heatmap_plot matrix.csv output --force
```

`analyze workflow` 常用选项：

- `--reference <name|index>`：参考物种名称或 1-based 索引。
- `--target-genes <id,id,...>`：目标基因 ID，用于 `reference_vs_targets`。
- `--up <n>` / `--down <n>`：目标基因上下游窗口大小，默认 20。
- `--threads <n>`：线程数。
- `--align-soft {blast,last,diamond_blastp}`：比对后端。
- `--dbtype {nucl,prot}`：序列类型。
- `--cscore <float>`：同源匹配过滤强度，默认 0.7。
- `--dist <int>`：共线性锚点间最大基因距离，默认 20。
- `--iter <int>`：block 过滤迭代次数，默认 1。
- `--min-block-size <int>`：最小 block 大小。
- `--formats svg,pdf`：输出格式。
- `--params '<json>'`：JSON 对象，覆盖 method_config 字段。

### 3.2 可编排子模块

适合只调用单个能力、批量脚本或外部系统集成。每个子模块都有显式输入/输出端口，可通过 `--input-ports` 绑定。

```powershell
# 只跑 MCscan 同源搜索与 block 计算
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":"input"}' --output-dir output --force

# 只绘制点图（需要前置 anchors 产物）
GenomeLens.exe analyze submodule jcvi.graphics_dotplot `
  --input-ports '{
      "species_pair":"input",
      "anchors":"input/query__subject.anchors"
    }' --output-dir output --force

# 直方图
GenomeLens.exe analyze submodule jcvi.graphics_histogram `
  --input-ports '{"numeric_files":["numbers.txt"]}' --output-dir output --force

# 热图
GenomeLens.exe analyze submodule jcvi.graphics_heatmap `
  --input-ports '{"matrix_csv":"matrix.csv"}' --output-dir output --force
```

`analyze submodule` 常用选项：

- `--input-ports '<json>'`：必填，端口绑定 JSON 对象。
- `--output-dir <dir>`：必填，输出目录。
- `--input-dir <dir>`：当端口里的 `species_pair` 是物种名列表时，用于发现文件。
- `--params '<json>'`：JSON 对象，覆盖子模块参数。
- `--formats svg,pdf`：输出格式。
- `--force`：允许复用已有输出目录。

完整端口与参数表见 [`子模块手册.md`](子模块手册.md)。

## 4. 能力发现

```powershell
# 列出所有一站式工作流和子模块
GenomeLens.exe workflow list --json

# 只列工作流
GenomeLens.exe workflow list --kind one_stop --json

# 只列子模块
GenomeLens.exe workflow list --kind sub_module --json

# 查看单个能力
GenomeLens.exe workflow describe pairwise_synteny --json
GenomeLens.exe workflow describe jcvi.graphics_histogram --json

# 校验端口绑定
GenomeLens.exe workflow validate --submodule jcvi.graphics_histogram `
  --ports '{"numeric_files":["numbers.txt"]}' --json
```

## 5. 外部 JSON 请求

`analyze run` 读取稳定的 `AnalysisRequest` JSON，复用与 CLI 相同的执行路径：

```powershell
GenomeLens.exe analyze template mcscan > request.json
GenomeLens.exe analyze schema > analysis-request.schema.json
GenomeLens.exe analyze run request.json
```

最小示例：

```json
{
  "schema_version": 1,
  "kind": "analysis_request",
  "method": "mcscan",
  "task_kind": "one_stop",
  "one_stop_workflow_id": "pairwise_synteny",
  "input": { "mode": "auto_directory", "directory": "input" },
  "output": { "directory": "output", "force": true, "formats": ["svg"] },
  "config": {},
  "options": { "preset": "auto", "min_block_size": 1 },
  "method_config": {}
}
```

子模块示例：

```json
{
  "schema_version": 1,
  "kind": "analysis_request",
  "method": "mcscan",
  "task_kind": "sub_module",
  "sub_module_id": "jcvi.graphics_histogram",
  "port_bindings": { "numeric_files": ["numbers.txt"] },
  "input": { "mode": "method_specific" },
  "output": { "directory": "output", "force": true, "formats": ["svg"] }
}
```

完整字段说明见 [`AnalysisRequest JSON.md`](AnalysisRequest%20JSON.md)。

## 6. 配置文件

`config init` 生成：

- `genomelens.config.json`
- `jcvi.config.json`

`jcvi.config.json` 提供方式优先级：

1. `--jcvi-config <path>`
2. 输入目录下的 `jcvi.config.json`
3. 当前工作目录下的 `jcvi.config.json`

完整字段说明见 [`配置文件说明.md`](配置文件说明.md)。

## 7. 结果位置

```text
output/
  report/run_summary.json        # 顶层统一摘要
  inputs/analysis_request.json   # 归一化请求快照
  results/figures/               # 最终图件
  intermediate/jcvi/             # 引擎中间产物
  intermediate/local/            # 局部共线性中间产物
  intermediate/pairwise/         # 多物种 pairwise 子任务产物
  inputs/prepared/               # 预处理后的 BED/CDS
```

## 8. 当前限制

- 多物种总图使用默认 layout/seqids 与默认配色，尚未实现自动美化优化。
- 全局区块合并、排序、过滤与机器学习评分属于后续能力。
- `analyze run` 当前支持 `schema_version = 1` 的 `analysis_request`，首个稳定方法为 `mcscan`。
- `task_kind=composition` 的可视化编排执行尚未实现。
