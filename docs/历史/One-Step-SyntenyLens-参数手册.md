# JCVI One-Step SyntenyLens 插件参数手册与运行指南

> 基于 TBtools-II 的 JCVI 局部共线性分析一键化插件  
> 版本：Windows 版本 | 适用场景：目标基因局部共线性快速分析

---

## 一、插件概述

**JCVI One-Step SyntenyLens** 是围绕目标基因局部共线性分析制作的 TBtools 插件。它将 JCVI 中常用的四个分步流程（GFF_to_BED → Catalog → Compara_synteny → Graphics_synteny）封装到一个界面中，用户通过文件选择框、参数选项和输出目录即可完成从数据准备到最终绘图的全流程。

**核心价值**：
- 适合快速、稳定地完成常规局部共线性分析
- 保留中间文件，支持回流到分步插件进行精细调整
- 以参考物种目标基因为中心，自动截取上下游区域并绘制多物种共线性图

---

## 二、界面参数总览

插件界面分为 **7 大功能区域**，共 **18+ 项配置参数**：

| 区域 | 功能 | 包含参数 |
|------|------|----------|
| **① 参考物种输入** | 设定分析基准物种 | Reference FASTA/CDS/PEP, Reference GFF/BED |
| **② 目标物种输入列表** | 设定比对物种（可多物种） | Target FASTA/CDS/PEP List, Target GFF/BED List |
| **③ 目标基因与输出目录** | 设定分析中心与结果保存位置 | Target Gene IDs, Output Directory |
| **④ 比对参数** | 控制同源搜索与共线性筛选 | Type, CPUs, C-score, Distance, Iter |
| **⑤ 窗口范围** | 控制局部区域大小与展示方式 | Up/down genes, Split Targets, Label Targets |
| **⑥ 绘图设置** | 控制最终图片样式 | Output Format, Figure Size, DPI, Glyph Style, Glyph Color, Shade Style |
| **⑦ 开始运行** | 执行分析 | Start 按钮 |

## 三、输入参数详解

### 3.1 参考物种输入（Reference）

| 参数名 | 说明 | 输入要求 | 常见错误 |
|--------|------|----------|----------|
| **Reference FASTA/CDS/PEP** | 参考物种的序列文件 | 基因组 FASTA、CDS FASTA 或蛋白 FASTA | 物种必须是目标基因 ID 所在的物种 |
| **Reference GFF/BED** | 参考物种的注释或坐标文件 | GFF/GFF3 格式，或已准备好的 BED | 若上框为 CDS/PEP，则此处必须对应 BED |

**两种输入模式**：

| 模式 | 第一个文件框 | 第二个文件框 | 特点 |
|------|-------------|-------------|------|
| **Genome + GFF 模式** | 基因组 FASTA | GFF 或 GFF3 | 从原始基因组注释开始，插件自动准备 BED 和 CDS/PEP |
| **CDS/PEP + BED 模式** | CDS FASTA 或蛋白 FASTA | BED | 使用已整理好的代表转录本，ID 完全可控 |

> **代表转录本提取规则**（仅 Genome + GFF 模式触发）：
> 1. 读取 mRNA 层，以 mRNA ID 作为后续 BED 和序列文件的 ID
> 2. 读取 CDS 层，根据 Parent 属性归组到对应转录本
> 3. 按以下优先级选择代表转录本：
>    - ① CDS 序列更长
>    - ② 无内部终止密码子
>    - ③ CDS 片段数量更多
>    - ④ mRNA ID 字母顺序

### 3.2 目标物种输入列表（Target）

| 参数名 | 说明 | 输入要求 |
|--------|------|----------|
| **Target FASTA/CDS/PEP List** | 一个或多个目标物种的序列文件 | 多个文件顺序需与 Target GFF/BED List 完全对应 |
| **Target GFF/BED List** | 一个或多个目标物种的注释文件 | 第 1 个注释文件对应第 1 个序列文件，依此类推 |

### 3.3 目标基因与输出目录

| 参数名 | 说明 | 输入要求 | 注意事项 |
|--------|------|----------|----------|
| **Target Gene IDs** | 参考物种中的目标基因 ID | 多个基因用英文逗号分隔 | **必须是参考物种中的 ID**，非目标物种 ID |
| **Output Directory** | 分析结果输出目录 | 建议选择空文件夹或新建目录 | 避免不同参数结果混在一起 |

## 四、分析参数详解（同源搜索与共线性筛选）

| 参数名 | 类型 | 默认值/示例 | 功能说明 | 调整建议 |
|--------|------|------------|----------|----------|
| **Type** | 下拉选择 | `nucl` | 比对模式：`nucl`（核酸，适合 CDS）/`prot`（蛋白） | 物种关系较远时，蛋白模式更稳定 |
| **CPUs** | 整数 | `16` | 使用的线程数 | 主要影响速度，不改变生物学结果；配置允许时设为 8 或 16 |
| **C-score** | 浮点数 | `0.7` | 同源匹配过滤强度 | 数值越高越严格，保留关系更少但更可靠；越低噪声越多 |
| **Distance** | 整数 | `20` | 共线性锚点之间允许的最大基因距离 | 数值越大，允许锚点间隔越远；越小对局部连续性要求越严格 |
| **Iter** | 整数 | `1` | Block 过滤迭代次数 | 数值越大，筛选越严格；默认适合先看结果，太乱时适当调高 |

> **参数影响范围**：Type、C-score、Distance、Iter 共同决定"哪些同源关系能进入 block"，它们不是单纯的美化参数。建议先用默认值跑通，再根据 dotplot、anchors 和局部图判断是否需要调整。

## 五、目标区域参数详解（窗口范围）

| 参数名 | 类型 | 默认值/示例 | 功能说明 | 效果对比 |
|--------|------|------------|----------|----------|
| **Up/down genes** | 整数 | `20` | 目标基因上游和下游各取多少个基因 | 数值越小越聚焦（如 10），数值越大区域越宽（如 30） |
| **Split Targets** | 复选框 | 未勾选 | 输入多个目标基因时，是否分开绘图 | 勾选：每个目标基因单独出图；不勾选：同染色体基因合并为连续区域 |
| **Label Targets** | 复选框 | 未勾选 | 是否在图中标注目标基因名称 | 目标基因少时建议打开；图拥挤或排版时建议关闭 |

**Up/down genes 效果**：
- `= 10`：更聚焦，仅显示核心邻域
- `= 30`：区域更宽，能看到更多上下游背景，但基因更密集

**Split Targets 效果**：
- **不勾选**：同一染色体上的多个目标基因合并为一个连续区域，适合查看基因间是否处于同一共线性背景
- **勾选**：每个目标基因分别作为中心单独出图

**Label Targets 效果**：
- **打开**：图中显示目标基因 ID（红色高亮连线）
- **关闭**：图面更干净，仅保留红色高亮，适合论文配图

## 六、绘图参数详解

| 参数名 | 类型 | 默认值/示例 | 选项/说明 | 选择建议 |
|--------|------|------------|-----------|----------|
| **Output Format** | 下拉选择 | `pdf` | `pdf` / `png` / `svg` | PDF 适合后期编辑；PNG 适合快速预览；SVG 适合矢量图工作流 |
| **Figure Size** | 文本 | `10x5` | 画布比例和大小（宽 x 高） | 基因多或物种多时加宽，如 `12x5`、`14x6` |
| **DPI** | 整数 | `300` | 图片分辨率 | 主要影响 PNG 清晰度；公众号建议不低于 200，正式出图可用 300 |
| **Glyph Style** | 下拉选择 | `box` | `box`（长方形）/ `arrow`（箭头） | box 更简洁；arrow 显示方向，适合强调基因方向 |
| **Glyph Color** | 下拉选择 | `orientation` | `orientation`（按链方向）/ `orthogroup`（按同源组） | orientation 着色方向；orthogroup 强调同源对应关系 |
| **Shade Style** | 下拉选择 | `curve` | `curve`（曲线）/ `line`（直线） | curve 更柔和，适合展示对应关系；line 更直接，密集图中更利落 |

**Glyph Style 对比**：
- **box**：长方形，视觉简洁
- **arrow**：箭头形状，直观显示基因链方向

**Glyph Color 对比**：
- **orientation**：蓝色/绿色等表示链方向（正链/负链）
- **orthogroup**：颜色强调同源组关系，对应基因使用相同/相近颜色

> **注意**：选择 `box` 不等于所有基因变为同色，颜色规则仍由 `Glyph Color` 决定。

**Shade Style 对比**：
- **curve**：曲线连接上下轨道，视觉更柔和
- **line**：直线连接，在密集图中更利落

## 七、运行流程（Pipeline）

插件将 JCVI 的四个分步流程整合为**一键式 Pipeline**：

```
┌─────────────────────────────────────────────────────────────────────┐
│ Step 1: 数据准备 (GFF_to_BED)                                        │
│ 输入: Genome + GFF / CDS + BED                                      │
│ 处理: 自动提取 BED、CDS/PEP，选择代表转录本，保证 ID 一致              │
│ 输出: prefix.bed, prefix.cds / prefix.pep                           │
└─────────────────────────────────┬───────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────┐
│ Step 2: 同源比对 (LAST)                                              │
│ 处理: nucl → lastal / prot → lastal                                  │
│ 输出: .last, .last.filtered, .anchors, .lifted.anchors, dotplot      │
└─────────────────────────────────┬───────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────┐
│ Step 3: 正交基因筛选 (JCVI Catalog)                                   │
│ 处理: 根据 C-score、Distance 等参数筛选同源关系                       │
│ 输出: 过滤后的 anchors 文件                                          │
└─────────────────────────────────┬───────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────┐
│ Step 4: 共线性区块构建 (MCscan / Compara_synteny)                      │
│ 处理: 根据锚点构建共线性 blocks，Iter 控制过滤强度                     │
│ 输出: .blocks 文件（文件名含 i1 等迭代标记）                           │
└─────────────────────────────────┬───────────────────────────────────┘
                                  │
┌─────────────────────────────────▼───────────────────────────────────┐
│ Step 5: 局部区域截取与绘图 (Graphics_synteny)                        │
│ 处理: 以目标基因为中心，截取 Up/down genes 范围，生成局部 blocks      │
│       自动生成 layout 文件，调用绘图引擎                              │
│ 输出: 最终图片 (PDF/PNG/SVG)，局部 .blocks, .bed, .layout            │
└─────────────────────────────────────────────────────────────────────┘
```

**整体流程口诀**：
> 提取序列与注释 → LAST 比对 → JCVI Catalog (筛选正交基因) → MCscan (共线性锚点与区块构建) → 绘图输出 (格式、尺寸、DPI、基因样式、颜色和连线样式)

## 八、输出目录结构

运行完成后，输出目录生成 **3 个核心文件夹**：

```
Output Directory/
├── ortholog/          # 同源搜索与数据准备结果
│   ├── *.bed
│   ├── *.pep / *.cds
│   ├── *.last
│   ├── *.last.filtered
│   ├── *.anchors
│   ├── *.lifted.anchors
│   └── dotplot 图     # 判断两物种整体同源关系是否正常
│
├── mcscan/            # 全基因组共线性区块
│   └── *.blocks       # 文件名含 i1 对应 Iter 参数
│
└── local/             # 局部共线性绘图结果（最终图片）
    ├── *.blocks       # 局部区块文件
    ├── *.bed          # 局部坐标文件
    ├── *.layout       # 轨道布局文件（可精修）
    └── *.pdf / *.png / *.svg   # 最终图片
```

| 文件夹 | 核心文件 | 用途 |
|--------|----------|------|
| **`ortholog/`** | `.bed`, `.pep/.cds`, `.last`, `.anchors`, dotplot | 检查数据准备和整体同源关系是否正常 |
| **`mcscan/`** | `.blocks` (含 `i1` 迭代标记) | 查看全基因组层面的共线性区块 |
| **`local/`** | 局部 `.blocks`, `.bed`, `.layout` + 最终图片 | **最终出图位置**；如需精修，用这三个文件导入 Graphics_synteny |

## 九、Layout 文件与精修

One-Step 插件的定位是快速得到可靠结果，但不会锁定版式。如需精修，可使用 `local/` 中的三个核心文件回流到 **Graphics_synteny** 插件：

| 文件 | 在 Graphics_synteny 中的位置 | 作用 |
|------|---------------------------|------|
| 局部 `.blocks` | Blocks / Synteny block 文件输入框 | 决定哪些基因之间画共线性连线 |
| 局部 `.bed` | BED 文件输入框 | 提供基因坐标和方向信息 |
| 局部 `.layout` | Layout 文件输入框 | 决定轨道位置、颜色、标签和连线关系 |

**Layout 轨道编号规则**：
- `# edges` 之前的每一行对应一条轨道，按出现顺序从 **0** 开始编号
- 轨道 0：参考物种目标区域（通常居中）
- 轨道 1：第一个目标物种
- 轨道 2：第二个目标物种（以此类推）

**Edges 写法推荐**：
- **参考中心式**（推荐）：`e, 0, 1` + `e, 0, 2` —— 所有目标物种直接与参考物种连线
- **相邻链式**：`e, 0, 1` + `e, 1, 2` —— 逐层相邻比较，但可能不够直观

**常见精修思路**：
1. 轨道太挤 → 拉开不同轨道的 `y` 值（如 0.80, 0.50, 0.20）
2. 物种名称太长 → 直接修改 `label` 为简短名称
3. 颜色不适合 → 修改 `color`（支持十六进制如 `#fc8d62`）
4. 连线太乱 → 检查 `# edges` 连接关系
5. 样式不满意 → **优先改 `.layout`**，不要改 `.blocks`

## 十、推荐使用方式

### 10.1 第一次分析
1. 使用 **Genome + GFF 模式**
2. 采用**默认参数**（C-score=0.7, Distance=20, Iter=1, Up/down=20）
3. 确认目标基因 ID 正确、ID 与注释文件对应
4. 查看 `local/` 中的输出图和 `ortholog/` 中的 dotplot

### 10.2 需要精细控制
- 先用分步插件或自定义脚本整理出 **CDS/PEP + BED**
- 将准备好的文件输入 One-Step，跳过自动提取步骤
- 或直接使用分步插件进行每一步的单独控制

### 10.3 正式出图
- 根据图面拥挤程度调整 **Up/down genes**（聚焦 vs 宽视野）
- 调整 **Figure Size**（基因/物种多时加宽）
- 调整 **Label Targets**（拥挤时关闭标签）
- 调整 **Glyph Style/Color** 和 **Shade Style** 匹配排版需求
- 如需矢量编辑，选择 **SVG** 格式；如需直接插入文档，选择 **PDF** 或高 DPI **PNG**

### 10.4 多物种分析
- **必须固定一个参考物种**，所有目标物种与之比较
- 推荐选择研究重点最明确、注释质量最好、目标基因 ID 最清楚的物种作为 Reference
- 多物种图以参考物种目标基因为中心组织所有轨道

## 十一、常见误解澄清

| 误解 | 正解 |
|------|------|
| 目标基因可以填目标物种的 ID | **错误**。Target Gene IDs 必须是**参考物种**中的 ID |
| 蛋白文件必须是 `.pep` 后缀 | **错误**。后缀不重要，关键是内容类型与 Type 选择一致（`.pep`, `.faa`, `.fa` 均可） |
| 蓝色和绿色长方形是不同类型基因 | **错误**。选择 `orientation` 时，颜色表示链方向；形状由 Glyph Style 决定 |
| 多物种分析可以随意拼图 | **错误**。必须固定参考物种，否则没有统一坐标框架，图会混乱 |
| 选择 `box` 后所有基因同色 | **错误**。颜色规则由 Glyph Color 决定，与形状无关 |
| One-Step 会取代分步插件 | **错误**。One-Step 适合常规快速分析；分步插件适合复杂控制、手动改 block/layout |
