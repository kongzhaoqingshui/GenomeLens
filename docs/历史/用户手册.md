# GenomeLens 用户手册

> 面向最终用户的快速参考。开发细节见 `docs/开发手册/README.md`。

## 1. 快速开始

GenomeLens 主要入口是 `analyze mcscan jcvi`，一条命令完成从输入发现到出图的全链路：

```powershell
GenomeLens.exe analyze mcscan jcvi input output --force
```

- `input`：输入目录，按物种 basename 自动发现文件对。
- `output`：输出目录。
- `--force`：允许复用或覆盖已有输出目录。

需要 JSON 摘要时加 `-j`：

```powershell
GenomeLens.exe analyze mcscan jcvi input output --force -j
```

## 2. 两种使用模式

### 2.1 一站式工作流

适合最常见的"从输入到出图"场景：

```powershell
GenomeLens.exe analyze mcscan jcvi input output --force
```

多物种（3 个及以上）输入会自动拆成 pairwise 子任务，并在顶层汇总为全局核型总图。

### 2.2 可编排子模块

适合只需要单个能力、批量脚本或外部系统集成的场景：

```powershell
# 只生成点图
GenomeLens.exe analyze mcscan jcvi input output --jcvi-workflow graphics_dotplot --force

# 核型共线性图
GenomeLens.exe analyze mcscan jcvi input output --jcvi-workflow graphics_karyotype --force

# 双向同源基因
GenomeLens.exe analyze mcscan jcvi input output --jcvi-workflow catalog_ortholog --force

# 直方图
GenomeLens.exe analyze mcscan jcvi graphics_histogram numbers.txt output --force

# 外部 JSON 请求
GenomeLens.exe analyze run request.json

# 独立热图
GenomeLens.exe plot heatmap expr.csv heatmap-out --formats png --force
```

## 4. 输入目录

把同一物种的文件放在同一个目录，并保持 basename 一致。

### BED + CDS

```text
input/
  speciesA.bed
  speciesA.cds
  speciesB.bed
  speciesB.cds
```

也支持蛋白序列：`.pep`、`.pep.fa`、`.faa`。

### GFF + FASTA

```text
input/
  speciesA.gff3
  speciesA.fa
  speciesB.gff3
  speciesB.fa
```

同一目录可以按物种混用两类输入。若同一物种同时提供两类文件，自动发现会优先使用 `BED + CDS/PEP`。

## 5. 常用参数

- `-c, --config <path>`：GenomeLens 主配置。
- `--jcvi-config <path>`：JCVI 配置；优先级高于位置参数。
- `--reference <name|index>`：参考物种名称或 1-based 索引。
- `--threads <n>`：线程数。
- `--align-soft {blast,last,diamond_blastp}`：比对后端。
- `--dbtype {nucl,prot}`：序列类型。
- `--cscore <float>`：同源匹配过滤强度。
- `--dist <int>`：共线性锚点间最大基因距离。
- `--iter <int>`：block 过滤迭代次数。
- `--min-block-size <int>`：最小共线性 block 大小。
- `--formats svg,pdf`：输出格式。
- `--jcvi-workflow <name>`：选择 workflow，例如 `graphics_synteny`、`graphics_dotplot`、`graphics_karyotype`、`local_synteny`、`catalog_ortholog`、`graphics_histogram`。

完整参数分组请查看：

```powershell
GenomeLens.exe help analyze mcscan
GenomeLens.exe help analyze mcscan io
GenomeLens.exe analyze mcscan jcvi --help
```

## 6. 目标基因局部共线性

指定 `--target-genes` 后自动进入 `local_synteny`：

```powershell
GenomeLens.exe analyze mcscan jcvi input output `
  --reference subject `
  --target-genes AT1G01010,AT1G01020 `
  --up 20 --down 20 `
  --force
```

常用参数：

- `--up <int>` / `--down <int>`：上下游基因数。
- `--split-targets`：每个目标基因单独出图。
- `--label-targets`：高亮目标基因标签。
- `--use-native-local-synteny-renderer`：使用原生 matplotlib 渲染器替代 JCVI 默认渲染器，支持跨染色体局部窗口；计算较重，默认关闭。

## 7. 外部 JSON 请求

`analyze run` 读取稳定的 `AnalysisRequest` JSON，复用与 `analyze mcscan jcvi` 相同的执行路径：

```powershell
GenomeLens.exe analyze template mcscan > request.json
GenomeLens.exe analyze schema > analysis-request.schema.json
GenomeLens.exe analyze run request.json
```

最小示例：

```json
{
  "schema_version": 2,
  "kind": "analysis_request",
  "method": "mcscan",
  "input": { "mode": "auto_directory", "directory": "input" },
  "output": { "directory": "output", "force": true, "formats": ["svg"] },
  "config": {},
  "options": { "preset": "auto", "min_block_size": 1 },
  "method_config": { "workflow": "graphics_synteny" }
}
```

完整字段说明见 [`使用方法/AnalysisRequest JSON.md`](使用方法/AnalysisRequest%20JSON.md)。

## 8. Histogram 直方图

```powershell
GenomeLens.exe analyze mcscan jcvi graphics_histogram numbers.txt output `
  --histogram-columns 0,1 `
  --histogram-bins 30 `
  --histogram-title "Histogram" `
  --force
```

常用参数：`--histogram-inputs`、`--histogram-columns`、`--histogram-skip`、`--histogram-bins`、`--histogram-vmin`/`--vmax`、`--histogram-xlabel`、`--histogram-title`、`--histogram-base`、`--histogram-facet`、`--histogram-fill`。

## 9. 独立绘图：Heatmap

```powershell
GenomeLens.exe plot heatmap expr.csv heatmap-out --formats png --force
```

输出目录复用标准 GenomeLens 布局：`inputs/`、`intermediate/jcvi/`、`results/figures/`、`report/run_summary.json`。

## 10. 配置文件

`config init` 生成：

- `genomelens.config.json`
- `jcvi.config.json`

`jcvi.config.json` 提供方式优先级：

1. `--jcvi-config <path>`
2. 位置参数（`output-dir` 之后）
3. 输入目录下的 `jcvi.config.json`
4. 当前工作目录下的 `jcvi.config.json`

完整字段说明见 [`使用方法/配置文件说明.md`](使用方法/配置文件说明.md)。workflow 与局部共线性链路说明见 [`使用方法/JCVI能力与配置.md`](使用方法/JCVI能力与配置.md)。

## 11. 结果位置

```text
output/
  report/run_summary.json        # 顶层统一摘要
  inputs/analysis_request.json   # 归一化请求快照
  results/figures/               # 最终图件
  intermediate/jcvi/             # 引擎中间产物
  intermediate/local/            # 局部共线性中间产物
  intermediate/pairwise/         # 多物种 pairwise 子任务产物
  inputs/prepared/               # 预处理后的 BED/CDS
```

## 12. 当前限制

- 多物种总图使用默认 layout/seqids 与默认配色，尚未实现自动美化优化。
- 全局区块合并、排序、过滤与机器学习评分属于后续能力。
- `analyze run` 当前支持 `schema_version = 2` 的 `analysis_request`，首个稳定方法为 `mcscan`。
