# GUI 视觉与交互风格指南

> 本指南面向 `gui/tauri/` 前端实现，定义 GenomeLens 桌面 GUI 的整体视觉方向、色彩、动效与首页布局。
>
> 负责前端的同学（成员 B）应以此为准搭建设计系统；其他成员在 Code Review 时对照本指南检查 UI 一致性。

---

## 1. 设计方向

- **现代化 + Web 感**：界面看起来像一款精致的 Web 应用，而不是传统桌面工具。
- **极简**：留白充足、信息层级清晰、避免过多装饰。
- **有品质的动效**：用细腻动画传递状态变化，但不喧宾夺主。
- **专业领域感**：首页与品牌元素应让用户一眼识别这是“比较基因组学分析工作台”。
- **冰蓝色调**：冷色、通透、科技感，符合生命科学 / 数据分析的心理预期。

### 2.5 色彩使用原则

- 背景以白/深蓝为主，冰蓝只作为**强调色**出现。
- 主要按钮用 `--ice-500`，hover 到 `--ice-400`，按下到 `--ice-600`。
- 错误/警告只在状态标签、提示条中使用，避免大面积红色。
- 深色模式下避免纯黑，用深蓝灰做底色，冰蓝更亮更通透。

## 3. 字体与排版

- **主字体**：系统无衬线字体栈 `"Inter", "PingFang SC", "Microsoft YaHei", sans-serif`。
- **等宽字体**：日志、路径、代码用 `"JetBrains Mono", "SF Mono", monospace`。
- **字号层级**：

| 用途 | 字号 | 字重 | 行高 |
|------|------|------|------|
| 页面大标题 | 32px | 700 | 1.2 |
| 区域标题 | 20px | 600 | 1.3 |
| 卡片标题 | 16px | 600 | 1.4 |
| 正文 | 14px | 400 | 1.6 |
| 辅助说明 | 12px | 400 | 1.5 |
| 标签 / Badge | 11px | 500 | 1 |

- 中文标题避免使用极细字重；最小正文行高 1.6，保证可读性。

---

## 4. 间距与圆角

- **基础单位**：4px。
- **常用间距**：4、8、12、16、24、32、48。
- **卡片内边距**：16px ~ 24px。
- **页面边距**：32px（大屏）/ 16px（小窗）。
- **圆角**：
  - 卡片 / 面板：`12px`
  - 按钮 / 输入框：`8px`
  - Badge / Tag：`9999px`（胶囊）
  - 弹窗：`16px`

### 5.2 设计要点

- **Hero 区**：
  - 大标题左对齐或居中，字号 32px，字重 700。
  - 副标题用 `--text-secondary`，字号 16px。
  - 背景可使用极淡的冰蓝渐变（`#F0F9FF` → `#FFFFFF`），或一层微妙的“基因组连接”装饰线（见第 8 节）。
- **主分析入口卡片**：
  - 3 张或 4 张大卡片横向排布，占据视觉中心。
  - 每张卡片顶部用冰蓝色线性图标，标题 16px，说明 14px，底部一个主按钮。
  - 卡片 hover 时轻微上浮 + 阴影加深 + 图标颜色变亮。
- **引擎与工具区**：
  - 与子引擎入口分开，用更紧凑的列表/条带形式呈现。
  - 当前可用引擎显示“已就绪”绿色小点；未启用引擎显示灰色“预留”。
- **最近项目**：
  - 列表形式，每行显示项目名称、方法、时间、状态标签。
  - 空状态时展示占位插画 + “暂无项目，点击新建分析”。

## 6. 组件风格

### 6.1 卡片（Card）

- 背景 `--surface`，圆角 12px，边框 1px `--border`。
- 阴影：`0 1px 3px rgba(0,0,0,0.05)`，hover 时 `0 8px 24px rgba(14,165,233,0.12)`。
- hover 动画：上移 2px，阴影加深，过渡 200ms ease-out。

### 6.2 按钮

| 类型 | 背景 | 文字 | Hover |
|------|------|------|-------|
| Primary | `--ice-500` | white | `--ice-400` + 微上浮 |
| Secondary | transparent | `--ice-600` | `--ice-50` 背景 |
| Ghost | transparent | `--text-secondary` | `--surface` 背景 |
| Danger | `--error/10` | `--error` | `--error/20` 背景 |

- 按钮统一圆角 8px，内边距 `8px 16px`。
- 点击时有 scale(0.98) 微反馈。

### 6.3 输入框与表单

- 背景 `--bg`，边框 `--border`，圆角 8px。
- Focus 时边框变为 `--ice-500`，并加 `0 0 0 3px rgba(14,165,233,0.15)` 光晕。
- 标签字号 12px，字重 500，颜色 `--text-secondary`。
- 错误态边框 `--error`，下方小字提示。

### 6.4 进度与状态

- 进度条：高度 6px，圆角 9999px，填充 `--ice-400` → `--ice-500` 渐变，背景 `--ice-100`。
- 运行中进度条可加 subtle shimmer 动画（从左到右的光条扫过）。
- 状态标签：胶囊形，文字 11px 大写，配色：
  - 运行中：`--ice-100` 背景 + `--ice-700` 文字
  - 成功：`#D1FAE5` 背景 + `#065F46` 文字
  - 失败：`#FEE2E2` 背景 + `#991B1B` 文字
  - 等待：`#FEF3C7` 背景 + `#92400E` 文字

### 6.5 日志流

- 等宽字体，字号 12px，行高 1.6。
- 背景 `--surface`，左侧 3px 边框 `--ice-200`。
- 自动滚动到底部，新行进入时有淡入动画。
- 错误行高亮：左边框变 `--error`，文字颜色变 `--error`。

## 7. 动效规范

### 7.1 原则

- 动画服务于**状态解释**和**愉悦感**，不拖慢操作。
- 优先使用 `transform` 和 `opacity`，避免触发布局抖动。
- 过渡时长通常 150ms–300ms，缓动 `cubic-bezier(0.4, 0, 0.2, 1)`。
- 复杂入场动画不超过 500ms。

### 7.2 常用动画

| 场景 | 动画 | 时长 |
|------|------|------|
| 页面入场 | 淡入 + 轻微上移（`opacity 0→1, translateY 8px→0`） | 300ms |
| 卡片 hover | 上移 2px + 阴影加深 | 200ms |
| 按钮点击 | scale 0.98 | 100ms |
| 列表项新增 | 高度展开 + 淡入 | 250ms |
| 进度条 shimmer | 渐变光条从左到右循环 | 1.5s infinite |
| 状态切换 | 颜色/图标淡入过渡 | 200ms |
| 模态框出现 | 背景淡入 + 内容从 0.95 缩放到 1 | 200ms |
| 日志新行 | 淡入 | 150ms |

### 7.3 首页装饰动效

Hero 区可加入一层极淡的**抽象共线性网络**背景动画：

- 用 SVG 或 Canvas 绘制若干小点（代表基因/染色体区段）和细线（代表共线性关系）。
- 点和线使用 `--ice-200` / `--ice-300`，透明度 0.2–0.4。
- 点做缓慢的布朗运动，线随点轻微摆动，整体不抢主体内容。
- 若用户开启“减少动画”，则暂停此背景动画。

## 8. 主题与可访问性

- 默认跟随系统主题，支持手动切换 Light / Dark。
- 所有颜色 token 必须提供 light/dark 两套值。
- 动效必须尊重 `prefers-reduced-motion`：复杂动画降级为简单淡入或直接显示。
- 对比度：正文与背景对比度 ≥ 4.5:1，大标题 ≥ 3:1。
- 焦点环：所有可交互元素必须有可见 focus 状态（冰蓝光晕）。

## 10. 检查清单

实现首页与核心界面时，请确认：

- [ ] 主色调严格使用冰蓝 token，无偏离品牌的突兀颜色。
- [ ] 首页 Hero 区明确展示“比较基因组学分析工作台”定位。
- [ ] 主分析入口与引擎/工具入口视觉层级分开。
- [ ] 卡片、按钮、输入框遵循统一圆角与阴影规范。
- [ ] 所有 hover/focus/click 有细腻反馈，但不夸张。
- [ ] 支持 Light / Dark 主题切换，token 不硬编码。
- [ ] 尊重 `prefers-reduced-motion`。
- [ ] 空状态、加载态、错误态都有统一设计，不空白。
