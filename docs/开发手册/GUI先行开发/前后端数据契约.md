# GUI 前后端数据契约草案

> 本文件记录 GUI Phase 0 的前后端通信契约草案。后续 B/C 在表单、状态机和结果解析上扩展时，应先更新本文件再落实现。

---

## 1. 通信原则

- GUI 前端通过 Tauri Command 调用 Rust 后端。
- Rust 后端通过子进程或 sidecar 调用 `genomelens` / `jcvi-genomelens`。
- GUI 不直接实现分析业务逻辑，不维护与 CLI 不一致的私有任务协议。
- 跨层数据使用可序列化 JSON 结构。
- 对于 `get_template()`、`get_analysis_schema()`、`check_environment()`、`read_summary()` 这类直接透传 platform CLI / JSON 文件的接口，**返回载荷保持平台既有字段命名**，不在 Tauri/Rust 层改写键名；当前 Phase 0/1 实际上以 `snake_case` 为准。
- GUI 本地表单状态、store、view model 可以使用 `camelCase`，但必须在边界层完成与平台 JSON 的双向转换。

## 2. Phase 0 Command

### `get_version()`

用于探测本地 platform CLI 与 JCVI engine 可用性。

```typescript
interface CommandVersion {
  ok: boolean;
  command: string;
  version: string;
  error?: string;
}

interface VersionInfo {
  platform: CommandVersion;
  engine: CommandVersion;
}
```

语义：

- `platform` 调用 `genomelens --version`。
- `engine` 调用 `jcvi-genomelens probe`。
- 当命令不存在或退出码非 0 时，`ok = false`，错误摘要写入 `error`。
- 该命令只做能力探测，不修改任何本地状态。

### `get_template(method)`

用于把平台侧 request 模板暴露给 GUI 表单。

```typescript
type GetTemplateInput = { method: "mcscan" | string };
type GetTemplateOutput = AnalysisRequest;
```

语义：

- Rust 后端调用 `genomelens analyze template <method>`。
- 当前先行版默认使用 `method = "mcscan"`。
- 返回值必须是平台 CLI 输出的 JSON，不在 GUI 后端重写模板默认值，也不改写原始 `snake_case` 字段名。

### `get_analysis_schema()`

用于把平台侧 AnalysisRequest JSON schema 暴露给 GUI 表单与验证层。

```typescript
type GetAnalysisSchemaOutput = Record<string, unknown>;
```

语义：

- Rust 后端调用 `genomelens analyze schema`。
- 返回值直接来自平台 CLI 的 JSON schema。
- GUI 可基于该 schema 做字段说明和动态表单校验，但不得修改平台 schema 语义。
- 若 GUI 侧需要 `camelCase` draft model，应在前端边界层自行映射，不额外派生第二套 platform schema。

## 3. 后续 V1 Command 草案

| Command | 输入 | 输出 | 说明 |
|---------|------|------|------|
| `check_environment` | 无 | check JSON | 复用 `genomelens check -j` |
| `list_projects` | `{ workspace: string }` | `ProjectSummary[]` | 列出工作区项目 |
| `create_project` | `{ workspace: string; name: string }` | `ProjectSummary` | 创建项目目录与元数据 |
| `run_analysis` | `{ requestPath: string; outdir: string }` | `RunHandle` | 启动分析进程 |
| `read_summary` | `{ outdir: string }` | `RunSummary` | 读取运行摘要 |
| `read_run_log` | `{ outdir: string; tailLines?: number }` | `RunLogSnapshot` | 读取日志 |
| `open_path` | `{ path: string }` | `void` | 打开系统文件管理器 |

## 4. 事件流草案

```typescript
type GuiEvent =
  | { name: "analysis:stdout"; payload: { line: string } }
  | { name: "analysis:state"; payload: { state: string; progress: number } }
  | { name: "analysis:finished"; payload: { status: "SUCCEEDED" | "FAILED" | "CANCELLED"; summary?: RunSummary } }
  | { name: "analysis:error"; payload: { message: string; code?: string; details?: Record<string, unknown> } };
```

后续状态机由成员 C 主导映射，成员 A 提供 Rust 事件推送能力，成员 B 负责展示组件。

## 5. Phase 1/2 GUI 稳定消费约定

本节约定的是 **GUI view-model 会稳定消费哪些字段**。这些约定不改变 Rust 传输层，不要求把 `snake_case` 改成 `camelCase`，也不要求把 `method_data` 在 Rust contract 层强类型展开。

### 5.1 `check_environment()` 稳定消费字段

GUI 环境诊断页 Phase 1 稳定消费以下字段：

- 顶层：`status`
- 工具块：`blastn`、`makeblastdb`、`magick`、`jcvi_engine`
- 每个工具块内部：`status`、`path`、`message`
- 可选补充：`install_attempts`、`engine_candidate_names`

### 5.2 `read_summary()` 顶层稳定消费字段

GUI 结果页 / 运行页 Phase 1/2 稳定消费以下顶层字段：

- `status`
- `schema_version`
- `workflow`
- `method`
- `task`
- `species`
- `final_figures`
- `artifact_index`
- `logs`
- `ui`
- `scoring`
- `analysis_request_path`

其中稳定使用的子字段包括：

- `ui.state`
- `ui.progress`
- `ui.primary_figures`
- `ui.summary_path`
- `ui.log_path`
- `logs.run_log`
- `artifact_index[].artifact_id`
- `artifact_index[].artifact_type`
- `artifact_index[].path`
- `artifact_index[].format`
- `artifact_index[].preview`
- `artifact_index[].metadata`

### 5.3 `method_data` 稳定消费字段

GUI 通过 view-model 读取 `RunSummary.method_data` 时，Phase 1/2 稳定消费以下键：

- `species_count`
- `pairing_strategy`
- `pairwise_jobs`
- `pairwise_job_count`
- `global_figures`
- `reference_name`
- `native_multi_species`
- `native_layout`
- `jcvi_backend`
- `jcvi_workflow`
- `simplified_fallback`
- `engine_summary_path`
- `blast_table`
- `anchors_path`
- `simple_path`
- `blocks_path`

其中 `pairwise_jobs[]` 稳定消费以下字段：

- `pair_id`
- `species_a_name`
- `species_b_name`
- `status`
- `outdir`
- `run_summary_path`
- `engine_summary_path`
- `blast_table`
- `anchors_path`
- `simple_path`
- `blocks_path`
- `query_bed`
- `subject_bed`
- `final_figures`
- `error`

### 5.4 GUI-owned Tauri command 命名约定

以下命令不是 platform JSON 直通载荷，Tauri command 输入/输出统一使用 `camelCase`：

- `list_projects`
- `create_project`
- `run_analysis`
- `read_run_log`
- `open_path`
