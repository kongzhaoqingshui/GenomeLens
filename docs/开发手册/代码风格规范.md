# GenomeLens 代码风格规范

> 本规范是 `开发规范.md` 的组成部分，聚焦 Python 代码格式与可读性。项目总纲、协作流程与发布规则请参见 `开发规范.md`。

本规范适用于 project-owned 代码，即：

- `platform/src/genomelens/`
- `engines/jcvi/src/jcvi_genomelens/`

不包含 vendored 的 JCVI 代码（`references/upstream/jcvi/`、`engines/jcvi/src/jcvi/` 等）。

自动化基线为 [Ruff](https://docs.astral.sh/ruff/)，配置见各模块的 `pyproject.toml`：

- `line-length = 120`
- `target-version = "py312"`
- lint 规则：`E`, `F`, `I`, `UP`, `B`

Ruff 无法覆盖的风格细节（region 注释、docstring 标点、空行语义）由本规范约束。

---

## 1. 文件头与模块 docstring

每个 Python 文件第一行必须是模块级 docstring，简述该文件职责

```python
"""CLI 展示层：集中管理颜色、字段和交互提示"""
```

- 单行即可，**末尾不加句号**。
- 需要时可在 docstring 里用括号标注中英术语，例如 `workflow(工作流)`。

---

## 2. Imports

### 2.1 必须包含 future annotations

```python
from __future__ import annotations
```

### 2.2 分组与顺序

1. 标准库
2. 第三方库
3. 项目内部模块

组与组之间空一行，组内按字母顺序排序（Ruff `I` 规则自动处理）。

### 2.3 用 region 包裹 import 段

```python
# region import
from __future__ import annotations

import argparse
import os

from genomelens.analysis.methods.registry import list_methods

# endregion
```

- `# region import` 后不加空行。
- 距离行尾空一行接 `# endregion`；`# endregion` 后空两行再接后续代码。

---

## 3. Region 注释

按功能把相关函数/类归到同一个 region 块中。

```python
# region 内部格式操作函数
def _supports_color(...) -> bool:
    ...


def _paint(...) -> str:
    ...

# endregion


# region 工作台与提示相关函数
def render_workbench_banner(...) -> str:
    ...

# endregion
```

格式要求：

- `# region 中文描述`：描述末尾**不加标点**。
- region 内函数之间空一行。
- `# endregion` 独占一行，前面空一行。
- `# endregion` 之后空两行再接下一个顶级定义（函数/类/region）。
- 避免 region 嵌套过深，一般只用于 `import`、同类工具函数、命令注册分组、帮助函数等。

---

## 4. Docstring

### 4.1 单行 docstring

用于简单函数/类，**末尾不加句号**：

```python
def _paint(text: str, color: str, *, enabled: bool) -> str:
    """按需为文本包裹 ANSI 颜色"""
```

### 4.2 多行 docstring

复杂函数使用多行，结构为：

```python
def run(manifest: EngineRunManifest, outdir: str | Path) -> tuple[list[CommandAudit], dict[str, object]]:
    """运行目标基因局部共线性分析

    步骤：
    1. 运行 pairwise MCscan 得到全基因组 blocks。
    2. 按目标基因上下游窗口截取局部 blocks。
    3. 生成局部 layout。
    4. 调用 JCVI graphics.synteny 出图。
    """
```

- 第一行是简短摘要，**不加句号**。
- 空一行后写详细说明。
- 详细说明内部可使用列表、段落。

### 4.3 中英术语

允许在 docstring 中用括号标注英文/中文术语，帮助跨背景读者：

```python
"""生成整理后的 analyze(分析) 文本摘要"""
```

## 5. 行注释

行注释解释“为什么”或“注意事项”，而非重复代码字面含义。

```python
# 边框内宽度 = max(显式最小宽度, 所有行的可见宽度)
inner = max([min_width, *(_visible_width(line) for line in lines)]) if lines else min_width

# 用户可以通过环境变量强制关闭/开启颜色
if os.environ.get("NO_COLOR"):
    return False
```

- 行尾注释与代码之间留 **两个空格**。
- 行注释末尾**不加句号**。
- 对复杂正则或状态机分支，推荐在逻辑段前加块注释说明。

---

## 6. 空行

### 6.1 顶级定义之间

函数与函数、类与类、region 与 region 之间空 **两行**。

### 6.2 region 内函数之间

空 **一行**。

### 6.3 函数内逻辑段之间

根据语义空行，把函数拆成“输入准备 → 核心计算 → 返回组装”等段落。例如：

```python
def render_check_report(payload: dict[str, object], *, color: bool | None = None) -> str:
    """生成整理后的 check(检查) 文本报告"""

    enabled = _supports_color() if color is None else color
    status = str(payload.get("status") or "unknown")
    status_color = PALETTE.green if status == "ok" else PALETTE.red

    lines = [
        "",
        f"{_paint('GenomeLens 环境检查', PALETTE.bold + PALETTE.blue, enabled=enabled)}",
        ...
    ]

    labels = {...}

    for key, label in labels.items():
        ...

    lines.append("")

    return "\n".join(lines)
```

## 7. 命名与类型

### 7.1 命名

- 函数/变量：`snake_case`
- 类：`PascalCase`
- 常量：模块级全大写 `SCREAMING_SNAKE_CASE`
- 私有辅助函数/变量：以下划线开头 `_helper`

### 7.2 类型提示

- 必须为主函数添加类型提示。
- 使用 `|` 联合类型：`str | None`、`list[str] | None`。
- 跨层 JSON 结构当前允许 `dict[str, object]`，但应逐步替换为 `TypedDict` 或 dataclass。

### 7.3 函数长度

- 目标：函数不超过 60 行。
- 超过 100 行的函数必须拆分或提取辅助函数。

---

## 8. 字符串与格式化

- 优先使用 f-string。
- 字符串引号：代码库统一使用双引号 `""`（Ruff format 会处理）。
- 多行字符串使用括号隐式连接或三引号 docstring，避免反斜杠续行。

---

## 9. 错误处理

- 捕获通用异常时，必须添加 `# noqa: BLE001` 与说明注释。
- CLI/引擎边界允许兜底捕获，但应转换为 `format_user_error` 等用户可读错误。
