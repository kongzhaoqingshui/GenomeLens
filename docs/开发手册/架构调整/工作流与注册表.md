# 工作流与注册表当前结构

当前平台公开入口是 `WorkflowRequest`，内部由 `WorkflowPlanner` 展开为 `ExecutionPlan`，再由 `PlanExecutor` 执行。旧的 `workflow/`、`analysis/methods/` 和 `app/controller/runners/` 目录已经移除。

## 代码位置

| 职责 | 当前路径 |
|---|---|
| workflow plugin 协议与注册表 | `platform/src/genomelens/analysis/workflows/registry.py` |
| workflow provider 协议 | `platform/src/genomelens/analysis/workflows/provider.py` |
| 内置 MCscan workflow | `platform/src/genomelens/analysis/workflows/builtin_mcscan.py` |
| 一站式 workflow 元数据 | `platform/src/genomelens/analysis/workflows/onestop.py` |
| submodule 元数据 | `platform/src/genomelens/analysis/workflows/submodules.py` |
| 输入端口与绑定 | `platform/src/genomelens/analysis/workflows/input_bindings.py` |
| 执行计划模型 | `platform/src/genomelens/analysis/planning/models.py` |
| 执行计划生成 | `platform/src/genomelens/analysis/planning/planner.py` |
| DAG 执行入口 | `platform/src/genomelens/analysis/execution/executor.py` |
| workflow step handler | `platform/src/genomelens/analysis/execution/handlers/` |

## 新增一站式 workflow

1. 在 `analysis/workflows/onestop.py` 注册 `OneStopWorkflowSpec`。
2. 在 `analysis/workflows/builtin_mcscan.py` 或对应插件中暴露该 workflow。
3. 在 `analysis/planning/planner.py` 增加展开规则。
4. 在 `analysis/execution/handlers/` 或 engine manifest builder 中补齐执行处理。
5. 更新 CLI 帮助、用户手册、能力接入规则和最小集成测试。

## 新增 submodule

1. 在 `analysis/workflows/submodules.py` 注册 `SubModuleSpec`。
2. 在 `analysis/workflows/input_bindings.py` 确认输入/输出端口类型。
3. 在 `analysis/execution/request_mapping.py` 或对应 handler 中完成请求映射。
4. 在 `engines/jcvi/src/jcvi_genomelens/workflows/dispatcher.py` 注册 engine workflow。
5. 更新 `docs/使用方法/子模块手册.md` 和最小测试。
