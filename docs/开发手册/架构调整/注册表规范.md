# 注册表规范

## 文件位置

- `platform/src/genomelens/workflow/submodule_registry.py`：所有可编排子模块。
- `platform/src/genomelens/workflow/onestop_registry.py`：所有一站式工作流。
- `platform/src/genomelens/workflow/port_system.py`：端口类型与端口声明抽象。

## 端口类型（port_kind）

| 类型 | 说明 | 运行时示例 |
|---|---|---|
| `species_pair` | 两个物种的输入对 | `"input"` 或 `["A","B"]` |
| `species_list` | 多个物种列表 | `["A","B","C"]` |
| `artifact` | 文件产物 | `"output/A__B.anchors"` |
| `value` | 任意 JSON 值 | `["AT1G01010"]` |
| `config` | 工具/环境配置 | `{"blastn":"..."}` |

## SubModuleSpec 字段

| 字段 | 必填 | 说明 |
|---|---|---|
| `module_id` | 是 | 全局唯一 ID，建议 `jcvi.<name>` |
| `name` | 是 | 显示名称 |
| `description` | 是 | 一句话说明 |
| `category` | 是 | 分类，如 `synteny_analysis`、`visualization` |
| `engine_workflow` | 是 | 映射到引擎 dispatcher 的 workflow 名称 |
| `standalone` | 是 | 是否可独立运行 |
| `inputs` | 是 | `PortDeclaration` 列表 |
| `outputs` | 是 | `PortDeclaration` 列表 |
| `parameters` | 是 | 子模块参数列表 |

## OneStopWorkflowSpec 字段

| 字段 | 必填 | 说明 |
|---|---|---|
| `workflow_id` | 是 | 全局唯一 ID |
| `name` | 是 | 显示名称 |
| `description` | 是 | 一句话说明 |
| `category` | 是 | 分类 |
| `runner` | 是 | 专用 runner 名称 |
| `engine_workflow` | 是 | 引擎 workflow 名称 |
| `equivalent_modules` | 否 | **仅用于文档/GUI 展示**，说明能力覆盖范围 |
| `optimization_notes` | 否 | 为什么不能简单用子模块拼接 |
| `parameters` | 是 | 工作流参数列表 |

## 注册原则

1. **子模块是最小能力单元**，输入/输出端口必须完整、无歧义。
2. **一站式工作流必须说明优化点**，并在 `runner` 中实现；不能只是子模块脚本。
3. **相同 engine_workflow 可被多个子模块复用**，但 `module_id` 必须唯一。
4. **参数默认值必须在注册表中声明**，CLI/GUI 自动读取并生成表单。
5. **新增注册必须同步**：
   - 引擎 dispatcher 映射（`SUBMODULE_ID_TO_WORKFLOW`）。
   - 平台 runner 路由（`submodule_runner.py` / `onestop_runner.py`）。
   - CLI 命令（如需要特殊参数）。
   - 文档（`子模块手册.md`、`用户手册.md`、`项目介绍.md`）。
   - 测试（单元测试 + 最小样例）。

## CLI/GUI 消费

```powershell
GenomeLens.exe workflow list --json
GenomeLens.exe workflow describe jcvi.graphics_dotplot --json
GenomeLens.exe workflow validate --submodule jcvi.graphics_dotplot --ports '{"species_pair":"input","anchors":"A__B.anchors"}' --json
```

GUI 基于 `workflow list` 和 `workflow describe` 渲染节点、端口和参数表单。
