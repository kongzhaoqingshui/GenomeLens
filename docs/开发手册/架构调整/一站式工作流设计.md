# 一站式工作流设计

## 定位

本文档定义 GenomeLens 1.X 中**一站式工作流（One-Stop Workflow）**的设计原则、与可编排子模块的区别，以及当前内置的一站式工作流清单。

## 设计原则

### 1. 不是子模块的简单拼接

一站式工作流面向最常见的分析场景，保留**独立、可优化**的执行路径。它虽然可以用子模块组合出近似结果，但平台为其提供专门 runner，允许：

- 中间产物复用与缓存（如避免重复预处理、复用 BED/CDS）。
- 并行调度优化（如 pairwise 子任务并行、图件并行渲染）。
- 场景专用参数默认值与自动推导（如 multi-species 自动选择参考物种）。
- 未来引入原生算法替换（如 Cython 加速、自研渲染器）。

### 2. 元数据透明

每个一站式工作流通过 `OneStopWorkflowSpec` 注册，记录：

- `workflow_id`：唯一 ID。
- `runner`：专用 runner 名称。
- `engine_workflow`：对应引擎层 workflow（可选）。
- `equivalent_modules`：等价的子模块组合（仅用于文档/GUI 展示）。
- `optimization_notes`：优化说明，明确其与组合路径的差异。

### 3. 与 Composable Sub-Module 的关系

| 维度 | One-Stop Workflow | Composable Sub-Module |
|---|---|---|
| 目标用户 | 研究者：一键完成某类分析 | 高级用户/开发者：自定义、复用、组合 |
| 执行路径 | 专用 runner，可优化 | 通用 runner，按端口连接执行 |
| 灵活性 | 低，按场景预设 | 高，可自由连接 |
| 性能 | 可针对场景加速 | 取决于组合方式 |
| GUI 呈现 | “分析向导”卡片 | 节点编辑器中的积木 |

## 当前内置一站式工作流

| workflow_id | 名称 | runner | 等价子模块组合 | 优化说明 |
|---|---|---|---|---|
| `pairwise_synteny` | Pairwise Synteny Analysis | `pairwise_runner` | `jcvi.mcscan_pairwise` → `jcvi.graphics_synteny` | 预计算 layout、合并 BED、并行 dotplot + synteny |
| `multi_species_synteny` | Multi-Species Synteny | `multi_species_runner` | `pairwise_synteny` (×N) → `jcvi.graphics_karyotype_global` | all-vs-all pairwise 并行 + 全局核型总图聚合 |
| `reference_vs_targets` | Reference vs Targets | `reference_vs_targets_runner` | `pairwise_synteny` (×N) → `jcvi.local_synteny_multi` | reference 与多个 target 的 pairwise + 局部共线性聚合 |
| `histogram_plot` | Histogram Plot | `histogram_runner` | `jcvi.graphics_histogram` | 纯 matplotlib 渲染，轻量 |
| `heatmap_plot` | Heatmap Plot | `plot_heatmap_command` | `jcvi.graphics_heatmap` | 直接消费矩阵 CSV |

## 注册表 API

```python
from genomelens.workflow.onestop_registry import get_onestop_registry

registry = get_onestop_registry()
registry.list_all()           # 全部一站式工作流
registry.get("pairwise_synteny")  # 单个规范
```

## 添加新的一站式工作流

1. 在 `platform/src/genomelens/workflow/onestop_registry.py` 中新增 `OneStopWorkflowSpec`。
2. 实现或复用专用 runner。
3. 在 CLI 中注册 `analyze workflow <workflow_id>`。
4. 更新本文档和 `docs/使用方法/` 中的用户手册。
