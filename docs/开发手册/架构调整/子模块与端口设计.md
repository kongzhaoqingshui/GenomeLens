# 子模块与端口设计

## 定位

本文档定义 GenomeLens 1.X 中**可编排子模块（Composable Sub-Module）**的元数据模型、端口类型与绑定规则，是 GUI 可视化编排和 CLI 能力发现的基础。

## 核心抽象

### SubModuleSpec

一个子模块的完整元数据，包含：

- `module_id`：全局唯一 ID，例如 `jcvi.graphics_dotplot`。
- `name` / `description`：展示名称与描述。
- `category`：分类，例如 `homology_search`、`synteny_analysis`、`visualization`。
- `engine_workflow`：映射到引擎层的 workflow 名称，例如 `graphics_dotplot`。
- `standalone`：是否可独立运行。
- `inputs` / `outputs`：`PortDeclaration` 列表。
- `parameters`：可调参数声明。

### PortDeclaration

端口声明：

- `port_id`：端口唯一标识。
- `port_kind`：端口数据类型，当前支持：
  - `species_pair`：两个物种的输入对。
  - `species_list`：多个物种列表。
  - `artifact`：文件产物（如 anchors、blocks、figures）。
  - `value`：普通值（如目标基因 ID 列表）。
  - `config`：配置对象（如工具链路径）。
- `required`：是否必填。
- `description`：描述。
- `artifact_type`：产物语义类型（如 `anchors`、`blocks`、`figures`）。
- `accepted_formats`：接受的文件扩展名列表。

## 端口绑定规则

运行时，`AnalysisRequest.port_bindings` 是一个 `dict[str, object]`，键为 `port_id`，值为运行时值。

`PortSystem.validate_bindings(inputs, port_bindings)` 会检查：

1. 所有 `required=True` 的输入端口必须被绑定。
2. 不允许出现声明之外的端口（防止拼写错误）。
3. 未来可扩展：检查 `port_kind` 与值类型是否匹配、文件是否存在、格式是否合法。

## 注册表 API

```python
from genomelens.workflow.submodule_registry import get_submodule_registry

registry = get_submodule_registry()
registry.list_all()           # 全部子模块
registry.get("jcvi.graphics_dotplot")  # 单个规范
```

## 未来扩展

- 在引擎层进一步拆分 `mcscan_pairwise`，可独立出 `jcvi.homology_search` 和 `jcvi.synteny_blocks`。
- 增加 `PortBinding` 的校验器插件，支持文件存在性、格式、物种名称一致性检查。
- 为 GUI 节点编辑器补充 `visual` 元数据（图标、颜色、位置）。
