# 配置文件说明

GenomeLens 把配置拆成两个 JSON 文件：

- `genomelens.config.json`：工具本身的运行设置。
- `jcvi.config.json`：JCVI 引擎与外部工具链的默认参数。

`config init` 会一次性写出这两个文件：

```powershell
GenomeLens.exe config init --workspace .work --force
```

执行后会在工作区生成：

```text
.work/
  genomelens.config.json
  jcvi.config.json
```

## 配置优先级

运行时按以下顺序解析，后生效的覆盖先生效的：

1. CLI 显式参数（如 `--threads`、`--blastn`）
2. 配置文件（`--config`、`--jcvi-config`）
3. 环境变量（`GENOMELENS_CONFIG`、`GENOMELENS_JCVI_CONFIG`）
4. 系统 `PATH`
5. 打包资源
6. `toolchains/` 本地缓存

## `genomelens.config.json` 字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `schema_version` | integer | 是 | `2` | 配置协议版本，当前为 2。 |
| `workspace_root` | string | 是 | `~/GenomeLensWork` | 运行时工作区根目录。 |
| `temp_root` | string | 是 | `<workspace_root>/temp` | 临时文件目录。 |
| `default_output_root` | string | 是 | `<workspace_root>/results` | 默认输出根目录。 |
| `log_level` | string | 否 | `INFO` | 日志级别，如 `DEBUG`、`INFO`、`WARNING`、`ERROR`。 |
| `jcvi_config_path` | string | 否 | `<workspace_root>/jcvi.config.json` | 关联的 JCVI 子配置文件路径。 |

示例（由 `config init --workspace .work` 生成）：

```json
{
  "schema_version": 2,
  "workspace_root": "D:\\myself\\GenomeLens\\.work",
  "temp_root": "D:\\myself\\GenomeLens\\.work\\temp",
  "default_output_root": "D:\\myself\\GenomeLens\\.work\\results",
  "log_level": "INFO",
  "jcvi_config_path": "D:\\myself\\GenomeLens\\.work\\jcvi.config.json"
}
```

## `jcvi.config.json` 字段

第 2 版配置按用途分组：`toolchain`、`runtime`、`mcscan`、`local_synteny`。当前仅支持第 2 版分组结构。

### `toolchain`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `jcvi_engine_path` | string | 否 | `""` | `jcvi-genomelens` 可执行文件显式路径；空字符串时自动定位。 |
| `blastn_path` | string | 否 | `""` | BLAST+ `blastn` 显式路径。 |
| `makeblastdb_path` | string | 否 | `""` | BLAST+ `makeblastdb` 显式路径。 |
| `lastal_path` | string | 否 | `""` | LAST `lastal` 显式路径。 |
| `lastdb_path` | string | 否 | `""` | LAST `lastdb` 显式路径。 |
| `magick_path` | string | 否 | `""` | ImageMagick `magick` 显式路径。 |

### `runtime`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `threads` | integer | 否 | `4` | 默认线程数。 |
| `formats` | list[string] | 否 | `["png"]` | 默认图件输出格式，如 `["png", "pdf"]`。 |

### `mcscan`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `workflow` | string | 否 | `graphics_synteny` | 默认 JCVI workflow(工作流)。 |
| `min_block_size` | integer | 否 | `5` | 默认最小共线性 block(区块) 大小。 |
| `align_soft` | string | 否 | `blast` | 默认比对后端：`blast` / `last` / `diamond_blastp`。 |
| `dbtype` | string | 否 | `nucl` | 默认序列类型：`nucl` / `prot`。 |
| `cscore` | float | 否 | `0.7` | 默认同源匹配过滤强度。 |
| `dist` | integer | 否 | `20` | 默认共线性锚点间最大基因距离。 |
| `iter` | integer | 否 | `1` | 默认 block 过滤迭代次数。 |
| `reference` | string | 否 | `""` | 默认参考物种名称或 1-based 索引；空字符串表示第一个物种。 |

### `local_synteny`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `target_gene_ids` | list[string] | 否 | `[]` | 目标基因 ID 列表。 |
| `up` | integer | 否 | `20` | 默认目标基因上游窗口大小。 |
| `down` | integer | 否 | `20` | 默认目标基因下游窗口大小。 |
| `split_targets` | boolean | 否 | `false` | 多个目标基因时是否各自单独出图。 |
| `label_targets` | boolean | 否 | `false` | 是否在图中标注目标基因名称。 |
| `glyphstyle` | string | 否 | `""` | 基因形状：`box` / `arrow`。 |
| `glyphcolor` | string | 否 | `""` | 基因着色：`orientation` / `orthogroup`。 |
| `shadestyle` | string | 否 | `""` | 连线样式：`curve` / `line`。 |
| `figsize` | string | 否 | `""` | 画布尺寸，例如 `10x5`。 |
| `dpi` | integer | 否 | `300` | 默认图件 DPI。 |

示例（由 `config init --workspace .work` 生成）：

```json
{
  "schema_version": 2,
  "toolchain": {
    "jcvi_engine_path": "",
    "blastn_path": "",
    "makeblastdb_path": "",
    "lastal_path": "",
    "lastdb_path": "",
    "magick_path": ""
  },
  "runtime": {
    "threads": 4,
    "formats": ["png"]
  },
  "mcscan": {
    "workflow": "graphics_synteny",
    "min_block_size": 5,
    "align_soft": "blast",
    "dbtype": "nucl",
    "cscore": 0.7,
    "dist": 20,
    "iter": 1,
    "reference": ""
  },
  "local_synteny": {
    "target_gene_ids": [],
    "up": 20,
    "down": 20,
    "split_targets": false,
    "label_targets": false,
    "glyphstyle": "",
    "glyphcolor": "",
    "shadestyle": "",
    "figsize": "",
    "dpi": 300
  }
}
```

## 指定配置文件

分析时可以通过 `--config` 和 `--jcvi-config` 显式指定：

```powershell
GenomeLens.exe analyze mcscan `
  -c .work\genomelens.config.json `
  --jcvi-config .work\jcvi.config.json `
  input output --force
```

也可以通过环境变量指定：

```powershell
$env:GENOMELENS_CONFIG = "D:\myself\GenomeLens\.work\genomelens.config.json"
$env:GENOMELENS_JCVI_CONFIG = "D:\myself\GenomeLens\.work\jcvi.config.json"
GenomeLens.exe analyze mcscan input output --force
```

## 路径留空 vs 显式指定

- 留空字符串 `""` 表示让 GenomeLens 按“CLI 参数 → 配置 → 环境变量 → PATH → 打包资源 → 本地缓存”的顺序自动定位。
- 显式写绝对路径会跳过自动搜索，直接命中该路径；如果文件不存在，运行时会报错。
- 工作区路径支持相对路径，`config init` 会将其解析为绝对路径写入文件。

## 常见配置场景

### 固定 BLAST+ 路径

```json
{
  "schema_version": 2,
  "toolchain": {
    "blastn_path": "C:\\Tools\\blast\\bin\\blastn.exe",
    "makeblastdb_path": "C:\\Tools\\blast\\bin\\makeblastdb.exe"
  },
  "runtime": {
    "threads": 8,
    "formats": ["png", "pdf"]
  },
  "mcscan": {
    "workflow": "graphics_synteny"
  }
}
```

### 多物种分析默认参数

```json
{
  "schema_version": 2,
  "runtime": {
    "threads": 8,
    "formats": ["png"]
  },
  "mcscan": {
    "min_block_size": 1,
    "workflow": "graphics_synteny"
  }
}
```

### 目标基因局部共线性默认参数

```json
{
  "schema_version": 2,
  "mcscan": {
    "align_soft": "last",
    "dbtype": "nucl",
    "cscore": 0.7,
    "dist": 20,
    "iter": 1
  },
  "local_synteny": {
    "target_gene_ids": ["AT1G01010", "AT1G01020"],
    "up": 20,
    "down": 20,
    "split_targets": false,
    "label_targets": true,
    "glyphstyle": "arrow",
    "glyphcolor": "orthogroup",
    "shadestyle": "curve",
    "figsize": "10x5",
    "dpi": 300
  }
}
```

## 注意事项

- `genomelens.config.json` 中只需要写工具级字段；JCVI 相关字段会被 `jcvi_config_path` 指向的文件覆盖。
- 如果同时存在 `genomelens.config.json` 和 `jcvi.config.json`，两者会合并，JCVI 文件中的同名字段优先级更高。
- 当前仅支持第 2 版分组结构，旧版扁平字段不再读取。
- 修改配置文件后不需要重新安装，下次运行 `analyze` 或 `check` 时会重新读取。
