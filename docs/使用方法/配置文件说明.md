# 配置文件说明

GenomeLens V3 把配置拆成两个 JSON 文件：

- `genomelens.config.json`：平台级配置，管理 workspace、工具链路径、runtime 默认值和 engine profile 链接。
- `jcvi.config.json`：JCVI engine profile，仅作为默认参数模板，不再承载任务身份、参考物种、目标基因或输入路径。

`config init` 会一次性写出这两个文件：

```powershell
GenomeLens.exe config init --workspace .work --force
```

执行后会在工作区生成：

```text
.work/
  genomelens.config.json   # 平台配置
  jcvi.config.json         # engine profile
```

## 配置优先级

运行时按以下顺序解析，后生效的覆盖先生效的：

1. CLI 显式参数（如 `--threads`、`--blastn`）
2. 请求 JSON 中的 `parameters` / `runtime`
3. engine profile（`jcvi.config.json`）中的默认参数
4. 平台配置（`genomelens.config.json`）中的工具链与 runtime 默认值
5. 环境变量（`GENOMELENS_CONFIG`、`GENOMELENS_ENGINE_PROFILE`）
6. 系统 `PATH`
7. 打包资源
8. `toolchains/` 本地缓存

## `genomelens.config.json` 字段

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `schema_version` | integer | 是 | `3` | 配置协议版本，当前为 3。 |
| `workspace_root` | string | 是 | `~/GenomeLensWork` | 运行时工作区根目录。 |
| `temp_root` | string | 是 | `<workspace_root>/temp` | 临时文件目录。 |
| `default_output_root` | string | 是 | `<workspace_root>/results` | 默认输出根目录。 |
| `log_level` | string | 否 | `INFO` | 日志级别，如 `DEBUG`、`INFO`、`WARNING`、`ERROR`。 |
| `engine_profile_path` | string | 否 | `<workspace_root>/jcvi.config.json` | 关联的 engine profile 路径。 |
| `toolchain` | object | 否 | 见下表 | 显式工具链路径。 |
| `runtime` | object | 否 | 见下表 | 通用 runtime 默认值。 |

### `toolchain`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `jcvi_engine_path` | string | 否 | `""` | `jcvi-genomelens` 可执行文件显式路径。 |
| `blastn_path` | string | 否 | `""` | BLAST+ `blastn` 显式路径。 |
| `makeblastdb_path` | string | 否 | `""` | BLAST+ `makeblastdb` 显式路径。 |
| `lastal_path` | string | 否 | `""` | LAST `lastal` 显式路径。 |
| `lastdb_path` | string | 否 | `""` | LAST `lastdb` 显式路径。 |
| `magick_path` | string | 否 | `""` | ImageMagick `magick` 显式路径。 |

### `runtime`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `threads` | integer | 否 | `4` | 默认线程数。 |
| `formats` | list[string] | 否 | `["svg"]` | 默认图件输出格式。 |

示例（由 `config init --workspace .work` 生成）：

```json
{
  "schema_version": 3,
  "workspace_root": "D:\\myself\\GenomeLens\\.work",
  "temp_root": "D:\\myself\\GenomeLens\\.work\\temp",
  "default_output_root": "D:\\myself\\GenomeLens\\.work\\results",
  "log_level": "INFO",
  "engine_profile_path": "D:\\myself\\GenomeLens\\.work\\jcvi.config.json",
  "toolchain": {
    "jcvi_engine_path": "",
    "blastn_path": "",
    "makeblastdb_path": "",
    "lastal_path": "",
    "lastdb_path": "",
    "magick_path": ""
  },
  "runtime": {
    "threads": 4,
    "formats": ["svg"]
  }
}
```

## `jcvi.config.json` 字段

V3 engine profile 是纯默认参数模板，按用途分组。已删除的旧字段包括：

- `mcscan.workflow`
- `mcscan.reference`
- `local_synteny.target_gene_ids`
- 任意输入路径或结构化聚合输入

### `synteny`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `align_soft` | string | 否 | `blast` | 默认比对后端：`blast` / `last` / `diamond_blastp`。 |
| `dbtype` | string | 否 | `nucl` | 默认序列类型：`nucl` / `prot`。 |
| `cscore` | float | 否 | `0.7` | 默认同源匹配过滤强度。 |
| `dist` | integer | 否 | `20` | 默认共线性锚点间最大基因距离。 |
| `iter` | integer | 否 | `1` | 默认 block 过滤迭代次数。 |
| `min_block_size` | integer | 否 | `5` | 默认最小 block 基因数。 |

### `local_synteny`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `up` | integer | 否 | `20` | 默认目标基因上游窗口大小。 |
| `down` | integer | 否 | `20` | 默认目标基因下游窗口大小。 |
| `split_targets` | boolean | 否 | `false` | 多个目标基因时是否各自单独出图。 |
| `label_targets` | boolean | 否 | `false` | 是否在图中标注目标基因名称。 |
| `use_native_local_synteny_renderer` | boolean | 否 | `false` | 是否使用原生 matplotlib 局部共线性渲染器。 |

### `plot`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `glyphstyle` | string | 否 | `""` | 基因形状：`box` / `arrow`。 |
| `glyphcolor` | string | 否 | `""` | 基因着色：`orientation` / `orthogroup`。 |
| `shadestyle` | string | 否 | `""` | 连线样式：`curve` / `line`。 |
| `figsize` | string | 否 | `""` | 画布尺寸，例如 `10x5`。 |
| `dpi` | integer | 否 | `300` | 默认图件 DPI。 |
| `auto_optimization.optimize_figsize` | boolean | 否 | `false` | 根据轨道数量自动推导合适画布尺寸。 |
| `auto_optimization.rewrite_layout_links` | boolean | 否 | `false` | 在全局核型/多物种局部图中按链式关系重写 layout edges。 |
| `auto_optimization.optimize_karyotype_labels` | boolean | 否 | `false` | 使用镜像 renderer 优化全局核型图轨道标签位置。 |

### `histogram`

直方图默认参数模板。实际调用通过 `SubmoduleRequest(module_id="jcvi.graphics_histogram")`。

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `inputs` | list[string] | `[]` | 默认输入文件列表。 |
| `columns` | list[int] | `[0]` | 默认读取列。 |
| `skip` | integer | `0` | 跳过行数。 |
| `bins` | integer | `20` | bin 数。 |
| `vmin` | number \| null | `0.0` | 最小值截断。 |
| `vmax` | number \| null | `null` | 最大值截断。 |
| `xlabel` | string | `"value"` | X 轴标签。 |
| `title` | string | `""` | 图标题。 |
| `base` | integer | `0` | 对数坐标底数。 |
| `facet` | boolean | `false` | 是否分面。 |
| `fill` | string | `"white"` | 柱体填充色。 |

### `heatmap`

热图默认参数模板。实际调用通过 `SubmoduleRequest(module_id="jcvi.graphics_heatmap")`。

| 字段 | 类型 | 默认值 | 说明 |
|------|------|--------|------|
| `matrix` | string | `""` | 默认矩阵 CSV 路径。 |
| `rowgroups` | string | `""` | 行分组文件路径。 |
| `cmap` | string | `""` | 默认 colormap。 |
| `groups` | boolean | `false` | 是否对列分组聚类。 |
| `horizontalbar` | boolean | `false` | 是否绘制顶部水平颜色条。 |

示例（由 `config init --workspace .work` 生成）：

```json
{
  "schema_version": 3,
  "synteny": {
    "align_soft": "blast",
    "dbtype": "nucl",
    "cscore": 0.7,
    "dist": 20,
    "iter": 1,
    "min_block_size": 5
  },
  "local_synteny": {
    "up": 20,
    "down": 20,
    "split_targets": false,
    "label_targets": false,
    "use_native_local_synteny_renderer": false
  },
  "plot": {
    "glyphstyle": "",
    "glyphcolor": "",
    "shadestyle": "",
    "figsize": "",
    "dpi": 300,
    "auto_optimization": {
      "optimize_figsize": false,
      "rewrite_layout_links": false,
      "optimize_karyotype_labels": false
    }
  },
  "histogram": {
    "inputs": [],
    "columns": [0],
    "skip": 0,
    "bins": 20,
    "vmin": 0.0,
    "vmax": null,
    "xlabel": "value",
    "title": "",
    "base": 0,
    "facet": false,
    "fill": "white"
  },
  "heatmap": {
    "matrix": "",
    "rowgroups": "",
    "cmap": "",
    "groups": false,
    "horizontalbar": false
  }
}
```

## 指定配置文件

分析时可以通过 `--config` 和 `--jcvi-config` 显式指定：

```powershell
GenomeLens.exe analyze workflow synteny `
  -c .work\genomelens.config.json `
  --jcvi-config .work\jcvi.config.json `
  input output --force
```

也可以通过环境变量指定：

```powershell
$env:GENOMELENS_CONFIG = "D:\myself\GenomeLens\.work\genomelens.config.json"
$env:GENOMELENS_ENGINE_PROFILE = "D:\myself\GenomeLens\.work\jcvi.config.json"
GenomeLens.exe analyze workflow synteny input output --force
```

旧环境变量 `GENOMELENS_JCVI_CONFIG` 仍被映射到 `GENOMELENS_ENGINE_PROFILE`，但新文档推荐使用后者。

## 路径留空 vs 显式指定

- 留空字符串 `""` 表示让 GenomeLens 按优先级自动定位。
- 显式写绝对路径会跳过自动搜索，直接命中该路径；如果文件不存在，运行时会报错。
- 工作区路径支持相对路径，`config init` 会将其解析为绝对路径写入文件。

## 常见配置场景

### 固定 BLAST+ 路径

```json
{
  "schema_version": 3,
  "toolchain": {
    "blastn_path": "C:\\Tools\\blast\\bin\\blastn.exe",
    "makeblastdb_path": "C:\\Tools\\blast\\bin\\makeblastdb.exe"
  },
  "runtime": {
    "threads": 8,
    "formats": ["svg", "pdf"]
  }
}
```

### 多物种分析默认参数

```json
{
  "schema_version": 3,
  "runtime": {
    "threads": 8,
    "formats": ["svg"]
  },
  "synteny": {
    "min_block_size": 1
  }
}
```

### 目标基因局部共线性默认参数

```json
{
  "schema_version": 3,
  "synteny": {
    "align_soft": "last",
    "dbtype": "nucl",
    "cscore": 0.7,
    "dist": 20,
    "iter": 1
  },
  "local_synteny": {
    "up": 20,
    "down": 20,
    "split_targets": false,
    "label_targets": true,
    "use_native_local_synteny_renderer": false
  },
  "plot": {
    "glyphstyle": "arrow",
    "glyphcolor": "orthogroup",
    "shadestyle": "curve",
    "figsize": "10x5",
    "dpi": 300,
    "auto_optimization": {
      "optimize_figsize": true,
      "rewrite_layout_links": true,
      "optimize_karyotype_labels": true
    }
  }
}
```

## 注意事项

- `genomelens.config.json` 只应包含平台级字段；分析参数请写入 `jcvi.config.json`。
- `jcvi.config.json` 是**默认参数模板**，不是第二套任务请求。它不能定义 `workflow_id`、`reference`、`target_gene_ids` 或输入路径。
- 当前仅支持第 3 版分组结构，旧版 `mcscan` / `local_synteny` 扁平或半扁平字段会由迁移 shim 自动转换并发出弃用警告。
- 修改配置文件后不需要重新安装，下次运行 `analyze` 或 `check` 时会重新读取。
