# 子模块手册

> 本手册列出 GenomeLens 1.X 所有可编排子模块的端口、参数与 CLI 示例。
> 可通过 `genomelens workflow describe <module_id> --json` 获取机器可读元数据。

当前 10 个子模块按 `module_kind` 分为两类：

- `lightweight`：输入是原始数据或轻量中间产物，例如 `species_pair`、`.anchors`、`.blocks`、矩阵 CSV、数值文件。
- `aggregate`：输入是 `tracks`、`edges`、聚合 `blocks`、merged BED 等构造式聚合输入，通常由平台聚合步骤或一站式 `synteny` 工作流自动准备。

## 约定

- **端口类型**：
  - `species_pair`：两个物种的输入，可以是目录路径，也可以是 `["speciesA", "speciesB"]` 物种名列表。
  - `artifact`：前置产物文件（如 `.anchors`、`.blocks`）。
  - `value`：普通值（如基因 ID 列表）。
  - `config`：工具链路径等配置。
- **参数覆盖**：通过 `--params '{"key":"value"}'` 传入。稳定字段会映射到 `parameters` 的对应分组，暂未强类型化的字段会进入 `parameters.extras`。

## 同源搜索

### `jcvi.mcscan_pairwise`

对两个物种执行 BLAST/LAST/Diamond 比对、锚点扫描与 block 计算。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种的 BED/CDS 或 GFF/FASTA 输入对 |
| `toolchain` | config | 否 | 比对工具链路径（blastn/makeblastdb/lastal/lastdb） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `blast_table` | artifact | BLAST 同源比对表（`.blast`） |
| `anchors` | artifact | 共线性锚点文件（`.anchors`） |
| `simple` | artifact | 简化共线性边文件（`.simple`） |
| `blocks` | artifact | 共线性 block 文件（`.blocks`） |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `align_soft` | string | `blast` | 比对后端 |
| `dbtype` | string | `nucl` | 序列类型 |
| `cscore` | number | 0.7 | 同源匹配过滤强度 |
| `dist` | integer | 20 | 共线性锚点间最大基因距离 |
| `iter` | integer | 1 | Block 过滤迭代次数 |
| `min_block_size` | integer | 1 | 最小 block 大小 |
| `threads` | integer | - | 线程数 |

```powershell
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":"input"}' `
  --output-dir output --force --params '{"cscore":0.8,"threads":8}'
```

### `jcvi.catalog_ortholog`

基于共线性 + RBH 识别双向同源基因。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `blast_table` | artifact | 比对结果表 |
| `anchors` | artifact | 锚点文件 |
| `ortholog` | artifact | 双向同源基因表 |

参数与 `jcvi.mcscan_pairwise` 相同。

```powershell
GenomeLens.exe analyze submodule jcvi.catalog_ortholog `
  --input-ports '{"species_pair":"input"}' --output-dir output --force
```

## 可视化

### `jcvi.graphics_dotplot`

基于锚点绘制两个物种的共线性点图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `anchors` | artifact | 是 | 共线性锚点文件（`.anchors`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `figsize` | string | `""` | 画布尺寸 |
| `dpi` | integer | 300 | 分辨率 |
| `formats` | array | `["svg"]` | 输出格式 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_dotplot `
  --input-ports '{
      "species_pair":"input",
      "anchors":"input/speciesA__speciesB.anchors"
    }' --output-dir output --force --params '{"formats":["svg","pdf"]}'
```

### `jcvi.graphics_synteny`

基于 blocks 与 layout 绘制多物种共线性对齐图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `blocks` | artifact | 是 | 共线性 block 文件（`.blocks`） |
| `layout` | artifact | 否 | JCVI layout 文件（`.layout`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `glyphstyle` | string | `""` | 基因形状 |
| `glyphcolor` | string | `""` | 基因着色 |
| `shadestyle` | string | `""` | 连线样式 |
| `figsize` | string | `""` | 画布尺寸 |
| `dpi` | integer | 300 | 分辨率 |
| `formats` | array | `["svg"]` | 输出格式 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_synteny `
  --input-ports '{
      "species_pair":"input",
      "blocks":"input/speciesA__speciesB.blocks"
    }' --output-dir output --force
```

补充说明：

- 只提供 `blocks` 时会直接绘制 synteny 主图，不再重复运行 BLAST/MCscan。
- 若同时提供 `anchors`，会额外输出 dotplot，并继续避免回退到 pairwise core。
- 若提供 `layout`，会优先使用给定 layout。

### `jcvi.graphics_karyotype`

绘制物种内或两物种核型共线性图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `blocks` | artifact | 是 | 共线性 block 文件 |
| `simple` | artifact | 否 | 简化共线性边文件 |
| `layout` | artifact | 否 | 核型图 layout 文件 |
| `karyotype_seqids` | artifact | 否 | 核型图 seqids 文件 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |
| `karyotype_seqids` | artifact | 核型图 seqids 文件 |
| `karyotype_layout` | artifact | 核型图 layout 文件 |

参数与 `jcvi.graphics_dotplot` 相同。

补充说明：

- 只提供 `blocks` 时，workflow 仍可运行，但可能在缺少 `simple` 与显式 `layout` 时回退重建 pairwise core。
- 若同时提供 `simple`，或显式提供 `layout` / `karyotype_seqids`，即可避免回退重算。

### `jcvi.graphics_karyotype_global`

> `module_kind = aggregate`。通常消费多物种 pairwise 结果聚合后的 `tracks` / `edges`，不建议普通用户手工拼装。

基于 tracks 与 edges 聚合全局核型总图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tracks` | value | 是 | 轨道列表（每个 track 含 name 与 bed 路径） |
| `edges` | value | 是 | 共线性边列表（每个 edge 含 i/j/simple 路径） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

参数与 `jcvi.graphics_dotplot` 相同。

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_karyotype_global `
  --input-ports '{
      "tracks":[
        {"name":"speciesA","bed":"A.bed"},
        {"name":"speciesB","bed":"B.bed"}
      ],
      "edges":[{"i":0,"j":1,"simple":"A__B.simple"}]
    }' --output-dir output --force
```

### `jcvi.graphics_histogram`

绘制数值分布直方图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `numeric_files` | artifact | 是 | 输入数值文件列表（`.txt`、`.tsv`、`.csv`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `histogram_columns` | array | `[0]` | 读取列号 |
| `histogram_bins` | integer | 20 | bin 数 |
| `histogram_vmin` | number | 0.0 | 最小值下界 |
| `histogram_vmax` | number | - | 最大值上界 |
| `histogram_xlabel` | string | `"value"` | X 轴标签 |
| `histogram_title` | string | `""` | 图标题 |
| `histogram_base` | integer | 0 | 对数坐标底数 |
| `histogram_facet` | boolean | false | 分面展示 |
| `histogram_fill` | string | `"white"` | 柱体填充色 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_histogram `
  --input-ports '{"numeric_files":["numbers.txt"]}' `
  --output-dir output --force --params '{"histogram_bins":40}'
```

## 局部共线性

### `jcvi.local_synteny`

以目标基因为中心绘制局部共线性图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `blocks` | artifact | 是 | 共线性 block 文件 |
| `target_genes` | value | 是 | 目标基因 ID 列表 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `up` | integer | 20 | 目标基因上游取基因数 |
| `down` | integer | 20 | 目标基因下游取基因数 |
| `split_targets` | boolean | false | 每个目标基因单独出图 |
| `label_targets` | boolean | false | 标注目标基因 |
| `use_native_local_synteny_renderer` | boolean | false | 使用原生 matplotlib 渲染器 |

```powershell
GenomeLens.exe analyze submodule jcvi.local_synteny `
  --input-ports '{
      "species_pair":"input",
      "blocks":"input/speciesA__speciesB.blocks",
      "target_genes":["AT1G01010"]
    }' --output-dir output --force --params '{"up":30,"down":30}'
```

补充说明：

- `target_genes` 可以通过端口直接传入，不必额外写入 request 的 `parameters.local_synteny.target_gene_ids`。
- 在提供 `blocks` 的情况下，workflow 会直接复用已有 pairwise core 产物生成局部图。

### `jcvi.local_synteny_multi`

> `module_kind = aggregate`。通常消费 reference-vs-targets 或多物种聚合步骤准备好的 `tracks`、聚合 `blocks`、merged BED 与目标基因列表。

在多个物种间以目标基因为中心绘制局部共线性总图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tracks` | value | 是 | 多物种轨道列表 |
| `blocks` | artifact | 是 | 共线性 block 文件 |
| `bed` | artifact | 是 | 合并后的 BED 文件 |
| `target_genes` | value | 是 | 目标基因 ID 列表 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

参数与 `jcvi.local_synteny` 相同。
