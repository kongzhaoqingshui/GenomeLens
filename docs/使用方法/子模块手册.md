# 子模块手册

> 本手册列出 GenomeLens v1.0.0-preview-1 所有可编排子模块的端口、参数与 CLI 示例。
> 可通过 `GenomeLens.exe workflow describe <module_id> --json` 获取机器可读元数据。

当前注册 **9 个子模块**。代码内部按三层分类器描述：

- **`category`**（职责分类，单值）：
  - `计算`：产出数据产物，不渲染。
  - `渲染`：纯图件渲染，不重新计算 pairwise。
  - `混合`：聚合/汇总型模块，通常消费多比较对或多物种构造式输入。
  - `其他`：预留扩展位，当前无模块。
- **`domain`**（领域分类，单值）：
  - `共线性`：依赖或产出共线性产物（anchors / blocks / simple / edges 等）。
  - `通用`：领域无关的通用绘图。
- **`labels`**（标签列表）：可挂载任意标签，供 CLI/GUI/插件按主题过滤。例如 `synteny_render`、`generic_plot`、`aggregate`、`multi_species`。

`module_kind` 仍保留作为编排类型：

- `lightweight`：输入是原始数据或轻量中间产物（`species_pair`、`.anchors`、`.blocks`、CSV/TXT 等）。
- `aggregate`：输入是构造式聚合数据（`tracks`、`edges`、merged BED），通常由平台聚合步骤或一站式 `synteny` 工作流自动准备。

---

## 约定

- **端口类型**：
  - `species_pair`：两个物种的输入，可以是目录路径，也可以是包含 `reference`/`target` 的字典。
  - `artifact`：前置产物文件（如 `.anchors`、`.blocks`、`.bed`）。
  - `value`：普通值（如基因 ID 列表、tracks/edges 列表）。
  - `config`：工具链路径等配置（一般通过 `runtime` 传入）。
- **参数覆盖**：通过 `--params '{"key":"value"}'` 传入。键名直接写入 `SubmoduleRequest.parameters`，按子模块契约声明的参数名读取。
- **请求文件**：除 CLI 外，子模块也可通过 `analyze run submodule_request.json` 调用，`inputs` 直接作为端口绑定，不要嵌套在 `inputs.ports` 下。

## 计算

### `jcvi.pairwise`（`lightweight`，`domain=共线性`）

对两个物种执行 BLAST/LAST/Diamond 比对、锚点扫描与 block 计算。`emit_ortholog=true` 时附带输出双向同源基因表（原 `jcvi.catalog_ortholog` 能力已合并进本模块）。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种的 BED/CDS 或 GFF/FASTA 输入对 |
| `toolchain` | config | 否 | 比对工具链路径（`blastn`/`makeblastdb`/`lastal`/`lastdb`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `blast_table` | artifact | BLAST 同源比对表（`.blast`） |
| `anchors` | artifact | 共线性锚点文件（`.anchors`） |
| `simple` | artifact | 简化共线性边文件（`.simple`） |
| `blocks` | artifact | 共线性 block 文件（`.blocks`） |
| `ortholog` | artifact | 双向同源基因表（仅 `emit_ortholog=true` 时产出） |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `align_soft` | string | `blast` | 比对后端 |
| `dbtype` | string | `nucl` | 序列类型 |
| `emit_ortholog` | boolean | `false` | 额外输出双向 ortholog 目录 |
| `cscore` | number | `0.7` | 同源匹配过滤强度 |
| `dist` | integer | `20` | 共线性锚点间最大基因距离 |
| `iter` | integer | `1` | Block 过滤迭代次数 |
| `min_block_size` | integer | `1` | 最小 block 大小 |
| `threads` | integer | `null` | 线程数（也可通过 `runtime.threads` 传入） |

```powershell
GenomeLens.exe analyze submodule jcvi.pairwise `
  --input-ports '{"species_pair":"input"}' `
  --output-dir output --force --params '{"cscore":0.8,"min_block_size":1}'
```

输出 ortholog 目录：

```powershell
GenomeLens.exe analyze submodule jcvi.pairwise `
  --input-ports '{"species_pair":"input"}' --output-dir output --force `
  --params '{"emit_ortholog":true}'
```

## 渲染

`category=渲染` 的模块**只画图，缺上游产物时报错**，不再静默重跑 pairwise。

### 共线性领域

#### `jcvi.graphics_dotplot`（`lightweight`）

基于锚点绘制两个物种的共线性点图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `anchors` | artifact | 是 | 共线性锚点文件（`.anchors`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `figsize` | string | `""` | 画布尺寸 |
| `dpi` | integer | `300` | 分辨率 |
| `formats` | array | `["svg"]` | 输出格式 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_dotplot `
  --input-ports '{
      "species_pair":"input",
      "anchors":"input/speciesA__speciesB.anchors"
    }' --output-dir output --force
```

#### `jcvi.local_synteny`（`lightweight`）

以目标基因为中心绘制局部共线性图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `species_pair` | species_pair | 是 | 两个物种输入对 |
| `blocks` | artifact | 是 | 共线性 block 文件（`.blocks`） |
| `target_genes` | value | 是 | 目标基因 ID 列表 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `up` | integer | `20` | 目标基因上游取基因数 |
| `down` | integer | `20` | 目标基因下游取基因数 |
| `split_targets` | boolean | `false` | 每个目标基因单独出图 |
| `label_targets` | boolean | `false` | 标注目标基因名称 |
| `use_native_local_synteny_renderer` | boolean | `false` | 使用原生 matplotlib 渲染器 |

```powershell
GenomeLens.exe analyze submodule jcvi.local_synteny `
  --input-ports '{
      "species_pair":"input",
      "blocks":"input/speciesA__speciesB.blocks",
      "target_genes":["AT1G01010"]
    }' --output-dir output --force --params '{"up":30,"down":30}'
```

### 通用领域

`histogram` / `heatmap` 是领域无关的通用绘图，不依赖任何 pairwise 产物。

#### `jcvi.graphics_histogram`（`lightweight`）

绘制数值分布直方图。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `numeric_files` | artifact | 是 | 输入数值文件列表（`.txt`、`.tsv`、`.csv`） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `histogram_columns` | array | `[0]` | 读取列号 |
| `histogram_bins` | integer | `20` | bin 数 |
| `histogram_vmin` | number | `0.0` | 最小值下界 |
| `histogram_vmax` | number | `null` | 最大值上界 |
| `histogram_xlabel` | string | `"value"` | X 轴标签 |
| `histogram_title` | string | `""` | 图标题 |
| `histogram_base` | integer | `0` | 对数坐标底数 |
| `histogram_facet` | boolean | `false` | 分面展示 |
| `histogram_fill` | string | `"white"` | 柱体填充色 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_histogram `
  --input-ports '{"numeric_files":["numbers.txt"]}' `
  --output-dir output --force --params '{"histogram_bins":40}'
```

## 混合

`category=混合` 的模块既不是纯计算也不是纯渲染，而是聚合/汇总型模块，通常消费多物种构造式输入。

### `jcvi.graphics_karyotype_global`（`aggregate`，`domain=共线性`）

基于 tracks 与 edges 聚合全局核型总图。通常消费多物种 pairwise 结果聚合后的 `tracks` / `edges`，不建议普通用户手工拼装。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tracks` | value | 是 | 轨道列表（每个 track 含 `name` 与 `bed` 路径） |
| `edges` | value | 是 | 共线性边列表（每个 edge 含 `i`/`j`/`simple` 路径） |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `figsize` | string | `""` | 画布尺寸 |
| `dpi` | integer | `300` | 分辨率 |
| `formats` | array | `["svg"]` | 输出格式 |

```powershell
GenomeLens.exe analyze submodule jcvi.graphics_karyotype_global `
  --input-ports '{
      "tracks":[
        {"name":"speciesA","bed":"A.bed"},
        {"name":"speciesB","bed":"B.bed"}
      ],
      "edges":[{"i":0,"j":1,"simple":"A__B.simple"}]
    }' --output-dir output --force
```

---

### `jcvi.local_synteny_multi`（`aggregate`，`domain=共线性`）

在多个物种间以目标基因为中心绘制局部共线性总图。通常消费 reference-vs-targets 或多物种聚合步骤准备好的 `tracks`、聚合 `blocks`、merged BED 与目标基因列表。

| 输入端口 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `tracks` | value | 是 | 多物种轨道列表 |
| `blocks` | artifact | 是 | 共线性 block 文件（`.blocks`） |
| `bed` | artifact | 是 | 合并后的 BED 文件（`.bed`） |
| `target_genes` | value | 是 | 目标基因 ID 列表 |

| 输出端口 | 类型 | 说明 |
|---|---|---|
| `figures` | artifact | 生成的图件 |

| 参数 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `up` | integer | `20` | 上游取基因数 |
| `down` | integer | `20` | 下游取基因数 |
| `split_targets` | boolean | `false` | 每个目标基因单独出图 |
| `label_targets` | boolean | `false` | 标注目标基因 |
| `use_native_local_synteny_renderer` | boolean | `false` | 使用原生渲染器 |

## 能力发现

```powershell
# 列出全部子模块（含 category/domain/labels/module_kind）
GenomeLens.exe workflow list --kind sub_module --json

# 查看单个模块完整元数据
GenomeLens.exe workflow describe jcvi.graphics_histogram --json
```

未来 CLI/GUI 可按 `category`、`domain` 或 `labels` 做过滤展示，无需改动模块注册表结构。
