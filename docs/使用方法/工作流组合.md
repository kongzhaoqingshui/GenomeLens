# 工作流组合

> 本手册面向高级用户、脚本编写者和 GUI 开发者，讲解如何通过子模块端口连接实现产物复用与低重复组合。

## 核心原则

1. **子模块组合，不是一站式拼接**
   - 一站式工作流内部有专用优化，不要把它拆成子模块再串起来。
   - 组合只发生在子模块之间：前一个子模块的输出端口作为后一个子模块的输入端口。

2. **显式端口绑定**
   - 每个子模块的输入/输出端口都在 [`子模块手册.md`](子模块手册.md) 中声明。
   - 组合时通过 CLI 的 `--input-ports` 或 GUI 连线把产物路径传递过去；平台请求里对应 `WorkflowRequest.inputs.ports`。

3. **产物复用**
   - 已经跑过的子模块产物可以复用，避免重复计算 BLAST/MCscan。

## 典型组合示例

### MCscan → dotplot

```powershell
# 1. 跑 MCscan，输出 anchors
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":"input"}' `
  --output-dir output/mcscan --force --params '{"threads":8}'

# 2. 用 anchors 绘制 dotplot
GenomeLens.exe analyze submodule jcvi.graphics_dotplot `
  --input-ports '{
      "species_pair":"input",
      "anchors":"output/mcscan/intermediate/jcvi/speciesA__speciesB.anchors"
    }' --output-dir output/dotplot --force
```

### MCscan → synteny

```powershell
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":"input"}' --output-dir output/mcscan --force

GenomeLens.exe analyze submodule jcvi.graphics_synteny `
  --input-ports '{
      "species_pair":"input",
      "blocks":"output/mcscan/intermediate/jcvi/speciesA__speciesB.blocks"
    }' --output-dir output/synteny --force
```

### pairwise × N → 全局核型总图

```powershell
# 三个物种 A、B、C
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":["A","B"]}' --input-dir input `
  --output-dir output/AB --force
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":["A","C"]}' --input-dir input `
  --output-dir output/AC --force
GenomeLens.exe analyze submodule jcvi.mcscan_pairwise `
  --input-ports '{"species_pair":["B","C"]}' --input-dir input `
  --output-dir output/BC --force

GenomeLens.exe analyze submodule jcvi.graphics_karyotype_global `
  --input-ports '{
      "tracks":[
        {"name":"A","bed":"input/A.bed"},
        {"name":"B","bed":"input/B.bed"},
        {"name":"C","bed":"input/C.bed"}
      ],
      "edges":[
        {"i":0,"j":1,"simple":"output/AB/intermediate/jcvi/A__B.simple"},
        {"i":0,"j":2,"simple":"output/AC/intermediate/jcvi/A__C.simple"},
        {"i":1,"j":2,"simple":"output/BC/intermediate/jcvi/B__C.simple"}
      ]
    }' --output-dir output/global --force
```

对于三个及以上物种，建议直接使用 `analyze workflow synteny` 一站式工作流，避免手动组合。

## 端口匹配规则

- **类型必须一致**：`artifact` 端口接文件路径，`value` 端口接 JSON 值，`species_pair` 端口接目录或物种名列表。
- **格式需兼容**：例如 `jcvi.graphics_dotplot` 的 `anchors` 端口需要 `.anchors` 文件。
- **必填端口必须全部绑定**，否则 `workflow validate` 会报错。

## 校验组合

```powershell
GenomeLens.exe workflow validate --submodule jcvi.graphics_dotplot `
  --ports '{
      "species_pair":"input",
      "anchors":"input/A__B.anchors"
    }' --json
```

## 与 GUI 的关系

未来 GUI 的可视化编排基于同一套端口元数据：

- 节点 = 子模块。
- 边 = 输出端口到输入端口的连接。
- 运行时生成 `WorkflowRequest`，并把节点输入归一化到 `inputs.ports` 或后续执行计划节点输入。

当前拖拽式 composition 执行尚未实现；脚本用户可先用分步 `analyze submodule` 命令达到同样效果。
