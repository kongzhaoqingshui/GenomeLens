# GenomeLens 项目进度汇报

> 汇报日期：2026-06-27  
> 当前版本：v1.0.0-preview-1  
> 本节拍拍：GUI 工作台骨架已落地，文档与仓库进入阶段性整理。

## 1. JCVI 引擎

### 已完成

- `engines/jcvi/` 作为独立分析引擎运行，暴露 `probe` 与 `run` 两个稳定入口。
- 已注册 9 个可调度 workflow：pairwise、graphics_dotplot、graphics_synteny、graphics_karyotype、local_synteny、local_synteny_multi、graphics_karyotype_global、graphics_histogram、graphics_heatmap。
- `catalog_ortholog` 能力合并进 `pairwise`，通过 `emit_ortholog=true` 输出双向直系同源目录。
- 引擎 manifest 与输出摘要 `engine_run_summary.json` 已升级到 v3。
- Windows 上 BLAST+ 路径兼容中文/部分特殊字符：pairwise `mcscan` 在 Windows 下将 `makeblastdb` / `blastn` 放到 ASCII 临时目录执行，再把结果移回目标目录。

### 进行中 / 待验证

- 与平台 shell 的多物种 all-vs-all pairwise 调度和全局总图汇总链路需持续集成测试。
- 全局 layout/seqids 自动美化仍由平台层负责，引擎本身不处理最终总图优化。

## 2. 平台核心

### 已完成

- CLI 入口统一为 `genomelens.cli.main`：
  - `analyze workflow synteny ...`
  - `analyze submodule <module_id> ...`
  - `analyze run <request.json>`
  - `analyze template workflow|submodule ...`
  - `analyze schema`
  - `workflow list / describe / validate`
  - `check`、`config init`、`project list/create`
- 公开任务协议收敛为 `WorkflowRequest v3`（仅 `synteny`）与 `SubmoduleRequest v3`（9 个子模块）。
- `WorkflowPlanner` 自动路由：2 物种 pairwise、≥3 物种多物种聚合、带 `target_gene_ids` 的 reference-vs-targets 局部共线性。
- `RunSummary v3` 含 `artifact_index`、`child_runs`、`extensions`，结果归档结构稳定。
- 配置拆分为平台配置 `genomelens.config.json` 与引擎 profile `jcvi.config.json`。

### 进行中 / 待补齐

- 运行事件流（`analysis:stdout/state/finished/error`）在 GUI 长链路中的稳定性需更多端到端验证。
- 子模块端口校验与产物类型提示可继续细化。

## 3. GUI（JCVI meow）

### 已完成

- 首页三栏布局：左侧能力搜索 + 列表、中心能力详情卡片、右侧对称占位。
- 工作台悬浮外壳 `WorkbenchShell`：左右侧栏浮动于画布之上，拖拽缩放不挤压画布。
- 左侧任务栏 `WorkbenchLeftPanel`：画布任务切换/移除、已保存任务、模板新建、设置入口。
- 节点画布基础设施：
  - `NodeCanvas`、`TaskNode`、`DataNode` 多端口渲染。
  - 数据节点编辑器 `DataNodeInspector`（species_pair / artifact / value）。
  - 图模型 `workbench-graph.ts`：拓扑排序、端口兼容、输入解析。
  - 任务预设持久化 `workbench-preset.ts`：任务关闭后仍可恢复，重启不丢失。
  - 串行运行引擎 `workbench-runner.ts`：拓扑执行、写入请求、调用 `runAnalysis`、回填输出端口。
- TypeScript 模型层已从旧 `AnalysisRequest v1` 迁移到 `WorkflowRequest v3` / `SubmoduleRequest v3`。

### 进行中 / 待补齐

- 右侧 Inspector 与画布节点的完整闭环：当前为占位状态，需把 `SyntenyForm` / `SubmoduleForm`、运行/结果视图按选中节点切换接入。
- 拖拽连线创建 `GraphEdge` 的交互已具备，但与运行引擎的端到端验证尚未跑通。
- 最小目标：串联 `species_pair 数据节点 -> jcvi.pairwise -> jcvi.graphics_dotplot / jcvi.graphics_synteny` 并实际运行出图。

## 4. 插件（HAIant / 智然体）

### 已完成

- 插件架构文档与参数映射文档已整理：`README.md`、`ARCHITECTURE.md`、`PARAMETER_MAPPING.md`。
- 一站式插件 `gljcvi-synteny` 与子模块插件（7 lightweight + 2 aggregate）已按 V3 协议定义输入/输出。
- 插件不实现分析算法，统一构造 `WorkflowRequest` / `SubmoduleRequest` 后调用外部 GenomeLens 可执行文件。

### 进行中 / 待补齐

- 插件包随平台发布流程的自动化打包与版本对齐。
- 插件表单与 GUI 能力选择器的字段同步，确保同一能力在 CLI/GUI/插件三端描述一致。

## 下一步重点

1. 完成 GUI 工作台右侧 Inspector 与节点编排的运行闭环。
2. 跑通 `jcvi.pairwise -> jcvi.graphics_dotplot / jcvi.graphics_synteny` 最小子模块链。
3. 补齐 GUI 端到端测试与运行事件稳定性测试。
4. 继续整理文档，确保历史归档与当前文档口径一致。
