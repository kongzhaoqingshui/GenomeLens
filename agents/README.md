# agents/ — 智能代理层（规划，尚未实现）

本目录预留给 GenomeLens 未来的 `AI Agent` 能力。按 `docs/开发手册/架构调整/最终架构目标.md` 第 5 节，Agent 是**平台服务层**，不是某个具体引擎的附属模块。

## 边界（重要）

Agent **通过平台任务接口工作**，不直接绕过平台执行底层脚本。可承担的职责：工作流规划、参数建议、输入质量检查、错误诊断、图件解释、结果对比、文献辅助、自动生成报告草稿。

约束：

- Agent 是调度者，不是绕过平台的脚本执行者。
- Agent 通过 `platform/` 暴露的稳定任务接口（`AnalysisRequest` / dispatcher）调度，而非直接拼 CLI 或调用引擎私有命令。

## 规划子目录（最终架构目标）

- `planner/`：工作流规划。
- `analyst/`：结果分析与解释。
- `reporter/`：报告草稿生成。

## 接入顺序

参见 `温和架构过渡.md`：Agent 与深度学习扩展排在接入顺序的最后，建在稳定平台接口之上。本目录现阶段仅作架构占位，不含实现代码。
